# O4O Shared Space Standard Blocks v1

> **연관 문서:** [O4O-SHARED-SPACE-FRAME-PRINCIPLE-V1.md](./O4O-SHARED-SPACE-FRAME-PRINCIPLE-V1.md)

## 1. 목적

이 문서는 Shared Space Frame 원칙에 따라, 공동 공간에서 사용할 수 있는 표준 블록을 정의한다.

공동 공간은 기능 실행 화면이 아니라, 여러 기능을 연결·요약·안내·진입시키는 공간이다.
따라서 이 문서의 목적은 다음과 같다.

* 공동 공간의 공통 블록 집합을 정의한다
* 어떤 블록이 필수인지, 선택인지, 제외 대상인지를 구분한다
* 서비스별 차별성을 허용하되 공간 문법은 유지하도록 한다
* 향후 Hub / Community Space / 유사 공간 설계 시 반복 사용 가능한 기준을 마련한다

---

## 2. 표준 블록 분류

공동 공간 표준 블록은 다음 3가지로 분류한다.

### 2.1 필수 블록

공동 공간이라면 원칙적으로 항상 검토해야 하는 블록

### 2.2 선택 블록

서비스 성격에 따라 포함 여부를 결정하는 블록

### 2.3 제외 블록

공동 공간 표준 블록으로 다루지 않는 블록

---

## 3. 필수 블록

### 3.1 Hero / Summary

#### 역할

* 이 공간이 무엇인지 설명한다
* 현재 사용자에게 중요한 내용을 요약한다
* 첫 행동을 유도한다

#### 포함 요소

* 제목 또는 환영 문구
* 짧은 설명
* 대표 CTA 1~3개

#### 금지

* 과도한 텍스트 설명
* KPI 중심 관리형 구성
* 너무 많은 버튼 배치

#### 비고

공동 공간은 기능 실행 화면이 아니라 맥락 제공 화면이므로, 상단 요약 블록은 필수로 본다.

---

### 3.2 News / Notices

#### 역할

* 공지
* 새소식
* 외부 정보
* 서비스 차원의 변화 안내

#### 포함 요소

* 공지 또는 뉴스 목록
* 탭 또는 그룹 구분 가능
* 전체보기 링크 가능

#### 허용

* 공지 / 새소식 / 외부 기사(예: 약사공론) 혼합 구성
* 서비스별 소스 차이 허용

#### 금지

* 지나치게 긴 본문 노출
* 기사 상세를 공동 공간에서 직접 소비하는 구조

#### 비고

공동 공간은 정보 허브 성격이 있으므로 News / Notices 블록은 필수로 둔다.

---

### 3.3 Activity

#### 역할

* 최근 활동
* 최근 글
* 최근 등록 콘텐츠
* 최근 변화

#### 포함 요소

* 최근 항목 목록
* 제목
* 날짜 또는 상태 정보
* 상세 이동 링크

#### 금지

* 너무 많은 메타 정보
* 테이블형 과밀 구성
* 관리형 운영 화면처럼 보이는 구성

#### 비고

이 블록은 공동 공간을 살아 있는 공간처럼 보이게 만드는 핵심 요소이므로 필수로 둔다.

---

### 3.4 App Entry

#### 역할

* 연결된 기능 앱으로 진입시킨다
* Forum / Content / Signage / LMS / 기타 기능으로 이동하게 한다

#### 포함 요소

* 카드 또는 진입 버튼
* 제목
* 짧은 설명
* 이동 액션

#### 금지

* 지나치게 많은 앱 나열
* 구조 설명 없이 아이콘만 배치
* 메뉴판처럼 과도하게 확장된 구성

#### 비고

공동 공간의 핵심 책임은 연결과 진입이므로 App Entry 블록은 필수다.

---

### 3.5 CTA / Guidance

#### 역할

* 다음 행동을 제안한다
* 참여를 유도한다
* 외부 연결 또는 운영 강조 액션을 보여준다

#### 포함 요소

* CTA 제목
* 설명
* 대표 버튼 또는 링크

#### 허용

* 외부 서비스 연결
* 운영자 강조 카드
* 캠페인성 액션

#### 금지

* 광고와 CTA를 혼동하는 구성
* 본문형 장문 설명

#### 비고

공동 공간은 단순 나열이 아니라 다음 행동을 제시해야 하므로 CTA / Guidance 블록을 필수로 둔다.

---

## 4. 선택 블록

### 4.1 Content Highlight

#### 역할

* 강의
* 추천 콘텐츠
* 큐레이션 콘텐츠
* 읽을거리 / 볼거리

#### 포함 요소

* 카드 또는 리스트
* 제목
* 요약 정보
* 전체보기 또는 상세 이동

#### 비고

콘텐츠가 핵심인 서비스에서는 중요하지만, 모든 공동 공간에 강제하지는 않는다.

---

### 4.2 Signage Preview

#### 역할

* 플레이리스트 미리보기
* 동영상 미리보기
* 매장 적용 가능한 시각 콘텐츠 노출

#### 포함 요소

* 썸네일
* 제목
* 재생시간 또는 구성 정보
* 상세 이동

#### 비고

Signage는 중요한 파트이지만 모든 공동 공간의 필수 요소는 아니므로 선택 블록으로 둔다.

---

### 4.3 Recommendation

#### 역할

* AI 추천
* 운영자 추천
* 개인화 추천
* 인기/추천 묶음

#### 포함 요소

* 추천 카드 또는 리스트
* 추천 이유 또는 간단 설명
* 이동 액션

#### 비고

중요한 고도화 요소이지만 초기 공통 필수 블록으로 강제하지는 않는다.

---

### 4.4 Utility

#### 역할

* 광고
* 후원사
* 외부 링크
* 배너
* 부가 안내

#### 포함 요소

* 링크형 카드, 배너, 보조 정보

#### 금지

* 공동 공간의 핵심 흐름을 방해하는 과도한 배치
* Hero/CTA보다 더 강한 강조

#### 비고

운영상 필요할 수 있으나 공동 공간의 본질은 아니므로 선택 블록으로 둔다.

---

## 5. 제외 블록

### 5.1 Dashboard KPI 블록

#### 제외 이유

* KPI 중심 구성은 공동 공간보다 대시보드에 가깝다
* 공동 공간을 관리형 포털처럼 보이게 만들 위험이 있다

---

### 5.2 편집 / 작성 블록

#### 제외 이유

* 공동 공간은 생성/편집 화면이 아니다
* 작성/편집은 기능 앱 내부 책임이다

---

### 5.3 상세 중심 블록

#### 제외 이유

* 공동 공간은 요약과 진입의 공간이다
* 긴 본문, 복잡한 상세 패널, 다단 편집형 구조는 공동 공간 표준 블록에 포함하지 않는다

---

## 6. 배치 원칙

공동 공간은 아래 흐름을 기본으로 한다.

```
1. Hero / Summary
2. News / Notices
3. Activity
4. App Entry
5. 선택 블록 (Content / Signage / Recommendation 등)
6. CTA / Guidance
7. Utility
```

### 배치 원칙

* 상단은 맥락과 요약
* 중단은 현재 활동과 기능 진입
* 하단은 선택 블록과 CTA/Utility
* 서비스별로 순서를 일부 조정할 수 있으나, 공동 공간의 흐름은 유지한다

---

## 7. 서비스별 차별성 허용 범위

공동 공간은 프레임과 블록 문법을 공유하되, 아래 차이를 허용한다.

* 블록 순서
* 강조 영역
* CTA 문구
* 용어
* 카드 타입
* 노출 밀도
* 외부 연결 요소

예:

* KPA는 News / Content 비중 강화
* GlycoPharm은 Guidance / Recommendation 비중 강화
* Neture는 CTA / App Entry 비중 강화

---

## 8. 적용 원칙

이 문서는 Hub / Community Space / 유사 공간 설계 시 우선 참고한다.

단,

* Forum core
* Signage core
* LMS core
* Content core

등 개별 기능 앱의 상세 구조를 대체하지 않는다.

즉,
기능 앱은 core + extension 구조를 유지하고,
공동 공간은 본 문서의 표준 블록을 조합하는 방식으로 설계한다.

---

## 9. 결론

공동 공간은 기능 실행 앱이 아니라, 여러 기능을 연결하고 안내하는 Shared Space Frame이다.

따라서 공동 공간의 표준화는 기능 단위가 아니라 블록 단위로 이루어져야 하며,
본 문서는 그 블록의 필수 / 선택 / 제외 기준을 정의한다.
