# O4O 플랫폼 — 앱(App) 단위 표준 목록 + 서비스 적용 매트릭스

> v1 · 2026-02-08

## 목적

* O4O는 서비스가 아니라 **앱(App)** 단위로 진화한다
* 앱은 **UI/UX/데이터 흐름의 표준 단위**
* 서비스는 **앱을 조합해서 구성**된다

---

## 1. O4O 앱(App) 표준 정의

> **앱(App)**
> UI 패턴 + UX 흐름 + 핵심 데이터 모델이 고정된 기능 단위
> (서비스와 무관, 재사용 가능)

---

## 2. O4O 표준 앱 목록 (고정)

| App ID       | 앱 이름       | 담당 데이터         | UI 기본형         |
| ------------ | ---------- | -------------- | -------------- |
| APP-CONTENT  | 콘텐츠 앱      | 공지, 뉴스, 배너, 혜택 | 카드 / 리스트       |
| APP-FORUM    | 포럼 앱       | 게시글, 댓글        | 카드 / 피드        |
| APP-SIGNAGE  | 디지털 사이니지 앱 | 영상, 플레이리스트     | Grid / Masonry |
| APP-LMS      | 학습 앱       | 강의, 퀴즈         | 카드 그리드         |
| APP-COMMERCE | 상품/주문 앱    | 상품, 주문         | 카드 ↔ 테이블       |
| APP-RESOURCE | 자료/리소스 앱   | 파일, 문서         | 리스트            |

> 이 목록은 늘릴 수는 있어도 쪼개지지 않는다.

---

## 3. 앱별 "표준화 범위"

모든 앱은 아래 항목을 **반드시 표준화**한다.

| 항목          | 설명                  |
| ----------- | ------------------- |
| UI 기본 패턴    | 카드 / 리스트 / 그리드 중 하나 |
| 정렬          | 최신 / 추천 / 조회        |
| 메인 요약 노출    | 메인화면에 요약 표시         |
| 출처 구분       | 운영자 / 공급자 / 사용자     |
| Empty State | 데이터 없음 처리           |
| Mobile 우선   | 모바일에서 성립            |

## 4. 서비스 ↔ 앱 적용 매트릭스

| 서비스         | 콘텐츠 | 포럼 | 사이니지 | LMS | 상품 |
| ----------- | --- | -- | ---- | --- | -- |
| KPA-Society | O   | O  | O    | P   | X  |
| Neture      | O   | O  | O    | X   | O  |
| GlycoPharm  | W   | X  | O    | X   | O  |
| K-Cosmetics | W   | X  | O    | X   | O  |

* O : 기본 포함
* W : 제한적 사용
* X : 미사용
* P : 예정

> 앱은 동일, 서비스는 조합만 다르다

---

## 5. 핵심 운영 원칙

### 원칙 A
**앱을 먼저 정비하고, 서비스를 나중에 적용한다**

### 원칙 B
**한 앱의 UI/UX 수정은 모든 서비스에 반영된다**

### 원칙 C
**서비스별 예외 UI는 허용하지 않는다 (필요하면 앱을 분리한다)**
