# **프론트엔드 개발 및 통합 지침서**

*Claude Code 작업 시 필수 준수사항*

## **🚨 핵심 원칙 (MUST FOLLOW)**

### **1\. 기존 코드베이스 절대 보호**

* **새로운 기능 추가 시 기존 기능을 절대 깨트리지 않는다**  
* 모든 기존 import, 컴포넌트, 서비스는 그대로 유지  
* 기존 라우팅 구조 변경 금지 (추가만 허용)

### **2\. 점진적 개발 원칙**

* 한 번에 모든 것을 바꾸지 않는다  
* 새로운 기능은 독립적인 모듈로 구현  
* 기존 시스템과 분리된 상태로 먼저 구현 후 통합

### **3\. 에러 전파 방지**

* 새로운 페이지의 에러가 전체 사이트에 영향주지 않도록  
* Error Boundary 적극 활용  
* 조건부 import 및 lazy loading 사용

## **📋 개발 전 필수 체크리스트**

### **작업 시작 전**

* \[ \] 현재 프로젝트 구조 파악 (`src/` 폴더 전체 스캔)

* \[ \] 기존 컴포넌트 의존성 맵핑  
* \[ \] package.json 의존성 확인

### **새로운 기능 개발 시**

* \[ \] 독립적인 폴더 구조로 격리 (`src/features/[feature-name]`)  
* \[ \] 기존 컴포넌트와 동일한 이름 사용 금지  
* \[ \] 전역 스타일 변경 최소화  
* \[ \] 기존 API 호출 패턴 준수

### **라우팅 추가 시**

* \[ \] 기존 라우트 유지하면서 새 라우트만 추가

* \[ \] 404 페이지 및 에러 핸들링 추가  
* \[ \] 브레드크럼 또는 네비게이션 일관성 유지

### **1단계: 격리된 개발**

src/  
├── features/  
│   └── healthcare/          \# 새로운 기능  
│       ├── components/  
│       ├── pages/  
│       ├── services/  
│       └── index.ts        \# 진입점  
├── shared/                 \# 기존 공통 컴포넌트  
└── App.tsx                \# 기존 라우팅 유지

### **2단계: 조건부 통합**

// ✅ App.tsx에서 안전한 라우팅 추가  
const HealthcareModule \= lazy(() \=\> import('./features/healthcare'));

function App() {  
  return (  
    \<Routes\>  
      {/\* 기존 라우트들 그대로 유지 \*/}  
      \<Route path="/existing-page" element={\<ExistingPage /\>} /\>  
        
      {/\* 새로운 기능은 조건부로 추가 \*/}  
      \<Route   
        path="/healthcare/\*"   
        element={  
          \<ErrorBoundary fallback={\<ErrorPage /\>}\>  
            \<Suspense fallback={\<Loading /\>}\>  
              \<HealthcareModule /\>  
            \</Suspense\>  
          \</ErrorBoundary\>  
        }   
      /\>  
    \</Routes\>  
  );  
}
