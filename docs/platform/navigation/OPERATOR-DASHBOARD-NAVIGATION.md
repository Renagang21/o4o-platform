# 운영자 대시보드 네비게이션 전략

> **문서 상태**: 기준 문서 (확정)
> **적용 범위**: 운영자 대시보드 UI 설계
> **관련 문서**: `EXTENSION-GENERAL-GUIDE.md`, `CONTENT-CORE-OVERVIEW.md`

---

## 1. 왜 네비게이션 전략이 필요한가

o4o 플랫폼은 다양한 Extension을 통해 기능이 확장된다.
각 Extension이 독자적인 네비게이션을 구현하면:

- 사용자 경험 불일치
- 학습 곡선 증가
- 유지보수 복잡도 증가

---

## 2. 네비게이션 원칙

### 2.1 통합 네비게이션

- 모든 Extension은 **통합 사이드바**에 메뉴를 등록한다
- 독자적인 네비게이션 구조를 생성하지 않는다
- 일관된 UI 패턴을 따른다

### 2.2 계층 구조

```
대시보드 (Dashboard)
├─ Core 메뉴
│  ├─ 콘텐츠 관리
│  ├─ 사용자 관리
│  └─ 시스템 설정
│
└─ Extension 메뉴
   ├─ LMS (Extension)
   │  ├─ 강좌 관리
   │  └─ 수강생 관리
   │
   └─ Signage (Extension)
      ├─ 디스플레이 관리
      └─ 콘텐츠 배치
```

---

## 3. 메뉴 등록 규칙

### Extension이 메뉴를 등록할 때

| 항목 | 규칙 |
|------|------|
| 위치 | Extension 영역 내에서만 |
| 아이콘 | 플랫폼 아이콘 세트 사용 |
| 라벨 | 한글 20자 이내 |
| 깊이 | 최대 2단계 |

### 금지 사항

- Core 메뉴 영역에 메뉴 삽입 ❌
- 3단계 이상 중첩 ❌
- 커스텀 아이콘 사용 ❌ (승인 필요)

---

## 4. 권한과 메뉴

### 권한 기반 메뉴 표시

- 사용자 권한에 따라 메뉴가 동적으로 표시/숨김
- Extension은 자신의 메뉴에 필요한 권한을 선언
- Core가 권한 체크를 수행

### 예시

```typescript
{
  menuId: 'lms-courses',
  label: '강좌 관리',
  path: '/admin/lms/courses',
  requiredPermissions: ['lms.courses.read']
}
```

---

## 5. 라우팅 규칙

### URL 구조

```
/admin/{extension}/{resource}
```

### 예시

| URL | 설명 |
|-----|------|
| `/admin/lms/courses` | LMS 강좌 목록 |
| `/admin/lms/courses/:id` | LMS 강좌 상세 |
| `/admin/signage/displays` | Signage 디스플레이 목록 |

### 규칙

- Extension prefix 필수 사용
- Core 라우트와 충돌 금지
- RESTful 패턴 권장

---

## 6. 상태 관리

### 네비게이션 상태

- 현재 활성 메뉴
- 펼침/접힘 상태
- 최근 방문 페이지

### 상태 공유

- Core가 네비게이션 상태를 관리
- Extension은 Core API를 통해 상태 조회
- Extension이 직접 상태를 수정하지 않음

---

## 7. 반응형 처리

### 브레이크포인트

| 크기 | 처리 |
|------|------|
| Desktop (≥1024px) | 사이드바 항상 표시 |
| Tablet (768-1023px) | 사이드바 축소 모드 |
| Mobile (<768px) | 햄버거 메뉴 |

### Extension 책임

- 각 브레이크포인트에서 자신의 페이지 반응형 처리
- 네비게이션 반응형은 Core가 담당
