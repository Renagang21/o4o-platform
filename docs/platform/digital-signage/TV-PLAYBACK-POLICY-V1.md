# TV Playback Policy V1

> Version: 1.0
> Date: 2026-01-19
> Status: Active
> Reference: ROLE-DEFINITION-V1.2

---

## 1. 문서 목적

이 문서는 Digital Signage가 **실제 TV/디스플레이 하드웨어**로 재생되는 시점의 **책임 경계**를 정의합니다.

Signage는 "웹 UI"가 아니라 **오프라인 TV**로 나가는 순간, 책임과 장애 원인이 **플랫폼/네트워크/매장 하드웨어**로 분기됩니다.

---

## 2. 재생 책임 경계

### 2.1 책임 분기표

| 구간 | 책임 주체 | 비고 |
|------|----------|------|
| 콘텐츠 선택·편성 | **약국(Store)** | Clone→Select→Arrange |
| 콘텐츠 제공 서버 | **플랫폼** | API, CDN, 스트리밍 |
| 실제 TV 재생 | **약국(Store)** | 하드웨어 동작 |
| 네트워크 환경 | **약국(Store)** | 인터넷 연결 |
| 강제 광고 콘텐츠 | **운영자/HQ** | isForced=true |

## 3. 하드웨어 전제 조건

### 3.1 약국 소유·관리 자산

다음 장비는 **약국 소유 및 관리 자산**입니다:

| 장비 | 소유 | 관리 책임 |
|------|------|----------|
| TV / 디스플레이 | 약국 | 약국 |
| 셋톱박스 / 미디어 플레이어 | 약국 | 약국 |
| 네트워크 장비 (공유기 등) | 약국 | 약국 |
| 인터넷 회선 | 약국 (계약자) | 약국 |

### 3.2 플랫폼 제공 범위

플랫폼은 다음을 **제공**합니다:

| 제공 항목 | 설명 |
|----------|------|
| 재생 앱/웹 플레이어 | Signage Player 소프트웨어 |
| 재생 URL | 매장별 고유 플레이어 URL |
| 콘텐츠 스트리밍 | CDN 기반 콘텐츠 전송 |
| API | 플레이리스트/스케줄 데이터 |

### 3.3 플랫폼이 보증하지 않는 것

플랫폼은 다음에 대해 **직접 보증하지 않습니다**:

- 약국 내 TV의 정상 동작
- 약국 네트워크 연결 상태
- 약국 플레이어 장치의 전원/상태
- 약국 환경으로 인한 재생 품질 저하

## 4. 장애 유형별 책임

### 4.1 장애 분류

| 장애 유형 | 원인 | 책임 주체 | 조치 |
|----------|------|----------|------|
| TV 전원 꺼짐 | 하드웨어 | **약국** | 약국 자체 점검 |
| TV 입력 소스 오류 | 설정 | **약국** | 약국 자체 점검 |
| 네트워크 끊김 | 인터넷 | **약국** | ISP 또는 공유기 점검 |
| 플레이어 미실행 | 소프트웨어/하드웨어 | **약국** | 플레이어 재시작 |
| 콘텐츠 로딩 실패 | 서버/CDN | **플랫폼** | 플랫폼 고객센터 |
| API 오류 | 서버 | **플랫폼** | 플랫폼 고객센터 |
| 강제 광고 미노출 | 시스템/설정 | **운영자/HQ** | HQ 문의 |
| 잘못된 콘텐츠 재생 | 편성 오류 | **약국** | 편성 재확인 |

### 4.2 문의 경로

| 장애 유형 | 문의처 |
|----------|--------|
| 하드웨어/네트워크 문제 | 약국 자체 또는 장비 공급업체 |
| 플랫폼 서버 문제 | 플랫폼 고객센터 |
| 강제 광고 관련 | HQ/운영자 |
| 편성 관련 | 약국 자체 관리 |

## 5. 오프라인 동작 정책

### 5.1 네트워크 단절 시

네트워크가 단절된 경우:

| 상황 | 동작 | 비고 |
|------|------|------|
| 캐시된 콘텐츠 있음 | 캐시 재생 | 제한적 동작 |
| 캐시된 콘텐츠 없음 | 재생 중단 | 오프라인 대기 화면 |
| 네트워크 복구 | 자동 재연결 | 최신 편성 반영 |

### 5.2 오프라인 재생 한계

- 오프라인 재생은 **캐시 범위 내**에서만 가능
- 실시간 콘텐츠/스트리밍은 오프라인 불가
- 네트워크 복구 시 **자동 동기화**

### 5.3 오프라인 책임

> 네트워크 단절로 인한 재생 중단은 **약국 책임**입니다.
> 플랫폼은 네트워크 복구 시 정상 재생을 지원합니다.

## 6. 플레이어 요구사항

### 6.1 지원 환경

| 환경 | 지원 여부 | 비고 |
|------|----------|------|
| 웹 브라우저 | 지원 | Chrome 권장 |
| Android TV | 지원 | 전용 앱 |
| 셋톱박스 | 지원 (호환 기기) | 사양 확인 필요 |
| 스마트 TV | 부분 지원 | 모델별 상이 |

### 6.2 최소 요구사항

| 항목 | 최소 요구 |
|------|----------|
| 인터넷 속도 | 10Mbps 이상 권장 |
| 메모리 | 2GB 이상 |
| 해상도 | 1920x1080 권장 |

### 6.3 권장 환경

- 유선 인터넷 연결 (Wi-Fi보다 안정적)
- 전용 미디어 플레이어 또는 Android TV
- 자동 전원 복구 설정

## 7. 약관 반영 문구

### 7.1 서비스 이용약관 반영

> **제X조 (Signage 하드웨어 책임)**
>
> 1. Digital Signage의 실제 재생에 필요한 TV, 셋톱박스, 미디어 플레이어, 네트워크 장비는 매장의 소유 및 관리 자산입니다.
>
> 2. 플랫폼은 콘텐츠 서버 및 재생 소프트웨어를 제공하며, 매장 하드웨어 및 네트워크 환경에 대해 보증하지 않습니다.
>
> 3. 매장 환경(하드웨어, 네트워크)으로 인한 재생 장애에 대해 플랫폼은 책임을 지지 않습니다.

### 7.2 면책 조항

> 플랫폼은 다음에 대해 책임을 지지 않습니다:
> - 매장 TV/디스플레이의 고장 또는 설정 오류
> - 매장 네트워크 환경으로 인한 재생 품질 저하
> - 매장 전원 공급 문제로 인한 재생 중단
> - 매장 환경으로 인한 콘텐츠 표시 왜곡

---

## 8. 고지 문구

### 8.1 플레이어 화면 고지

플레이어 설정 화면에 표시:

> **하드웨어 책임 안내**
>
> TV, 네트워크 장비 등 재생 환경은 매장에서 관리합니다.
> 재생 문제 발생 시 먼저 장비 및 네트워크 상태를 확인해 주세요.

### 8.2 장애 발생 시 안내

재생 오류 시 표시:

> **재생 오류**
>
> 콘텐츠를 불러올 수 없습니다.
> 1. 네트워크 연결을 확인해 주세요.
> 2. 플레이어를 재시작해 주세요.
> 3. 문제가 지속되면 고객센터로 문의해 주세요.
