## 1. 서론

### 1.1 문서의 목적

본 문서는 **서비스가 LMS를 연결해야 하는지** 판단하는 기준을 명확히 정의한다.

- LMS가 **반드시 필요한 조건** (MUST)
- LMS가 **선택적인 조건** (OPTIONAL)
- LMS를 **배제해야 하는 조건** (EXCLUDED)

이를 통해 서비스 기획 단계에서 LMS 연결 여부를 감정/사후논쟁 없이 즉시 판정할 수 있다.

### 1.2 왜 LMS 연결 기준이 필요한가

| 문제 | 해결책 |
|------|--------|
| "교육 기능 있으면 좋겠는데" | 명확한 판정 기준으로 결정 |
| LMS 남용 (불필요한 교육화) | EXCLUDED 조건으로 차단 |
| LMS 누락 (필수인데 없음) | MUST 조건으로 강제 |
| Forum/LMS 경계 혼란 | 역할 분리 명확화 |

### 1.3 문서의 지위

- 본 문서는 **CLAUDE.md**에 종속되며, 충돌 시 CLAUDE.md가 우선한다.
- **서비스별 기본 인프라 선언 문서 v1.0**과 연계하여 사용한다.
- **Forum 연결 기준 체크리스트 v1.0**과 상호 참조한다.
- LMS 관련 모든 개발은 본 문서의 기준을 따른다.

## 2. LMS Core 최소 책임 재확인

### 2.1 LMS Core가 하는 것 (MUST)

| 책임 | 설명 |
|------|------|
| 강좌 CRUD | 강좌/과정 생성/조회/수정/삭제 |
| 레슨 관리 | 강좌 내 개별 학습 단위 |
| 수강 등록 | 사용자-강좌 연결 |
| 진도 추적 | 학습 진행률 기록 |
| 퀴즈/평가 | 학습 성취도 측정 |
| 수료증 발급 | 과정 완료 인증 |
| 권한 제어 | 강사/수강생/관리자 역할 |

### 2.2 LMS Core가 하지 않는 것 (MUST NOT)

| 금지 | 이유 |
|------|------|
| Forum 컨텐츠 자동 승격 | 비즈니스 로직 |
| Signage 자동 배포 | Signage Core 책임 |
| 커리큘럼 자동 생성 | Extension 책임 |
| AI 학습 추천 | v1 범위 외 |
| 라이브 스트리밍 | 외부 서비스 연동 영역 |

> LMS는 **"구조화된 학습 경험을 제공"**할 뿐,
> **"무엇을 가르칠지"**는 결정하지 않는다.

### 2.3 Forum vs LMS 경계

| 구분 | Forum | LMS |
|------|-------|-----|
| 목적 | 의견 수집/토론 | 체계적 학습 |
| 구조 | 비정형 (게시물) | 정형 (강좌→레슨→퀴즈) |
| 진행 | 자유 참여 | 순차적 진도 |
| 성과 | 활동량 | 수료/점수 |
| 주체 | 다자 간 | 강사→수강생 |

## 3. LMS 연결 판정 체크리스트

### 3.1 MUST (필수 연결) 조건

다음 중 **2개 이상** 해당되면 LMS 연결이 **필수**다.

| # | 체크 항목 | 예시 |
|---|-----------|------|
| □ | **법정 필수 교육**이 서비스에 포함 | 약사 보수교육 |
| □ | **자격/인증**이 서비스 핵심 가치 | 전문 자격 취득 |
| □ | **체계적 온보딩**이 서비스 품질에 필수 | 신규 회원 교육 |
| □ | **학습 진도 추적**이 운영에 필수 | 교육 이수 관리 |
| □ | 수료/평가 결과가 **권한/등급에 영향** | 자격 등급 연동 |

### 3.2 OPTIONAL (선택 연결) 조건

다음 중 **1개 이상** 해당되면 LMS 연결은 **선택**이다.

| # | 체크 항목 | 예시 |
|---|-----------|------|
| □ | 제품/서비스 **사용법 안내** | 사용 가이드 영상 |
| □ | 직원/파트너 **역량 강화** | 셀러 교육 |
| □ | 고객 **자가학습 지원** | FAQ 동영상 |
| □ | 마케팅용 **무료 강좌** | 브랜드 교육 컨텐츠 |

### 3.3 EXCLUDED (배제) 조건

다음 중 **1개라도** 해당되면 LMS를 **연결하지 않는다**.

| # | 체크 항목 | 이유 |
|---|-----------|------|
| □ | **개인 의료/건강 데이터** 기반 학습 | 개인정보보호 |
| □ | 교육이 **1회성 안내**로 충분 | LMS 과잉 |
| □ | **커뮤니티 토론**만 필요 (Forum 적합) | 목적 불일치 |
| □ | 사용자 수 **극소수** (10명 미만) | ROI 부적합 |
| □ | 학습 진도/수료가 **무의미** | 구조 불필요 |

## 5. Extension 설계 원칙

LMS를 연결하는 서비스는 **Extension**을 통해 확장한다.

### 5.1 Contract 사용 필수

Extension은 **LMSContractClient**를 통해서만 LMS Core에 접근한다.

```typescript
import { LMSContractClient } from '@o4o/lms-core';

const client = new LMSContractClient(baseUrl, authToken);
await client.courses.create({
  name: '2025년 약사 보수교육',
  categoryId: '...'
});
```

### 5.2 ownsTables 정책

| 권장 | 설명 |
|------|------|
| `ownsTables: []` | LMS Core 테이블만 사용 (권장) |
| `ownsTables: ['ext_*']` | Extension 전용 테이블이 필요한 경우 |

### 5.3 비즈니스 로직 캡슐화

Extension은 자체 Service로 비즈니스 로직을 캡슐화한다.

| Extension | 비즈니스 로직 예시 |
|-----------|-------------------|
| lms-yaksa | 보수교육 학점 관리, 자격 연동 |
| lms-cosmetics | BA 교육 과정, 브랜드 인증 |

## 8. 자동 연결 금지 정책

### 8.1 v1.0에서 명시적으로 금지하는 자동화

| 금지 항목 | 이유 |
|-----------|------|
| Forum → LMS 자동 승격 | 컨텐츠 품질 보장 불가 |
| LMS → Signage 자동 배포 | 비즈니스 판단 필요 |
| 수료 → 권한 자동 부여 | Extension 책임 |

> 모든 단계 간 연결은 **명시적 트리거**가 필요하다.

### 8.2 향후 확장 조건 (v2.0 이후)

- 자동 연결은 **별도 Work Order**로 정의
- 각 서비스별 **명시적 승인** 필요
- **감사 로그** 필수

## 부록 B: 체크리스트 요약 (Quick Reference)

### B.1 3초 판정

```
1. EXCLUDED 해당? → 배제
2. MUST 2개+?    → 필수
3. OPTIONAL 1개+? → 선택
4. 그 외          → 배제
```

### B.2 MUST 체크리스트 (2개 이상 = 필수)

- [ ] 법정 필수 교육이 서비스에 포함
- [ ] 자격/인증이 서비스 핵심 가치
- [ ] 체계적 온보딩이 서비스 품질에 필수
- [ ] 학습 진도 추적이 운영에 필수
- [ ] 수료/평가 결과가 권한/등급에 영향

### B.3 EXCLUDED 체크리스트 (1개라도 = 배제)

- [ ] 개인 의료/건강 데이터 기반 학습
- [ ] 1회성 안내로 충분
- [ ] 커뮤니티 토론만 필요
- [ ] 사용자 수 극소수
- [ ] 학습 진도/수료가 무의미
