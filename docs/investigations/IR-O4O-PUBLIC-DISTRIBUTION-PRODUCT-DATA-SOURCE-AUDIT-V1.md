# IR-O4O-PUBLIC-DISTRIBUTION-PRODUCT-DATA-SOURCE-AUDIT-V1

> 공공데이터 기반 **유통용 표준 상품 구조(대표상품 + 포장단위)** 후보 데이터 소스 조사
> 작성일: 2026-06-30 · 성격: read-only 웹 조사 (코드/DB/스크립트 작성 없음)

---

## 1. 목적

정부 공공데이터 중 O4O의 **유통용 표준 상품 구조** 후보를 생성하는 데 참고 가능한 데이터 소스를 발굴한다.

- **대표상품** = 콘텐츠/소비자 안내 기준 (사람이 인지하는 "이 제품")
- **포장단위** = 공급/유통/주문 기준 (실제 거래·물류 SKU)
- 목적은 **완성 상품DB 구축이 아니라** "후보 생성에 참고할 공공데이터 발굴"
- **빈 필드 허용.** 공급자/매장 직접 등록을 대체하려는 것이 아니라 초기 후보 시드를 줄 수 있는지를 본다.

## 2. 조사 범위

상품군 우선순위 (높은 순서대로 깊게 조사):
1. 의약품 2. 의약외품 3. 건강기능식품 4. 의료기기 5. 화장품 6. 약국 유통 가능 일반 소비재(식품/수입식품 등)

확인 항목: 상품명, 제조사/업체명, 브랜드명, 포장단위/수량/단위, 품목기준코드, 표준코드, 허가/신고번호, 바코드(또는 연결가능 코드), 품목구분, 상품군, 최소 상태값, 제공방식, 인증키 필요 여부, 호출제한, 갱신주기, 이용허락(상업적 활용).

## 3. 조사 제외 범위

본 조사는 **유통용**이다. 다음은 명시적으로 제외한다 (수집/판단/추천 대상 아님):

- DUR, 병용/연령/임부금기, 복약지도
- 처방/약물/질환 상담, 환자별 판단, 처방 적정성
- 효능·효과·용법·주의사항 등 임상/상담 텍스트 필드는 유통 구조 후보에 사용하지 않음

> 처방 품목(전문의약품 포함)도 "유통 대상 여부(전문/일반 구분)"만 본다.

## 5. 의약품 (우선순위 1 — 가장 적합)

### 5.1 식약처 의약품 제품 허가정보 (data.go.kr 15095677)
- **제공방식**: OpenAPI (JSON/XML), 실시간 갱신, 무료, 인증키 필요(개발 1만콜/일, 운영 확장)
- **이용허락**: "이용허락범위 제한 없음" → 상업적 활용 가능
- **주요 필드**: 품목명, 주성분, 제조원, **포장단위**, 저장방법, 성상, 허가일자, 허가번호, 제형, 규격, **품목기준코드**, **표준코드**, **전문일반구분**, 희귀의약품 여부, (취소/취하 상태)
- **평가**: 상품명+제조사+포장단위+품목기준코드+표준코드+허가상태를 단일 소스에서 모두 제공. **유통 구조 후보 생성에 가장 적합.**

### 5.2 심평원 약가마스터 의약품표준코드 (data.go.kr 15067462)
- **제공방식**: **CSV 파일 다운로드(로그인 불필요)** + OpenAPI. 약 **298,183행**. 갱신 연1회(차기 2026-11-30)
- **이용허락**: 공공저작물 제1유형(출처표시) → 상업적 활용 가능(출처표시 조건)
- **주요 필드**: 한글상품명, 업체명, 약품규격, **제품총수량**, 제형구분, **포장형태**, 품목기준코드, 품목허가일자, **표준코드(13자리 KD코드)**, 대표코드, 제품코드(개정후), 일반명코드, **국제표준코드(ATC)**, 특수관리약품구분
- **평가**: **키 없이 CSV로 일괄 확보 가능한 유일한 대형 의약품 소스.** 표준코드(13자리)는 그 자체가 GS1 GTIN-13/14 바코드의 근간 → 바코드 연결 키로 활용 가능. 1차 import 후보의 핵심.

### 5.3 식약처 묶음의약품정보서비스 (data.go.kr 15063908)
- **제공방식**: OpenAPI(JSON/XML), 실시간, 무료, 키필요, 제한없음
- **구조**: 동일 성분/유사 특성 의약품을 **대표(대표 품목기준코드·대표 제품명·대표 업소명·대표 ATC) + 멤버 품목** 묶음으로 제공
- **평가**: O4O의 **"대표상품 ↔ 포장단위/변형 그룹핑" 개념과 1:1로 대응.** 대표상품 후보 자동 생성에 직접 활용 가능. (단, 묶음 기준은 "성분 유사"이므로 O4O의 "동일 제품 다른 포장" 개념과는 결이 다를 수 있어 정제 필요.)

### 5.4 바코드/표준코드 메모
- 의약품 표준코드(KD코드)는 국가(3)+업체(4)+품목(5)+검증(1) = 13자리. GS1 GTIN-14, GS1-128, SSCC로 물류 바코드화. → **표준코드를 보유하면 바코드 연결 가능.**
- 별도 의약품 바코드 raw 데이터셋(유통바코드)은 식품 중심 + 2018 갱신중단이라 의약품에는 표준코드 경유가 정석.

## 8. 의료기기 (우선순위 4)

### 식약처 의료기기 표준코드별 제품정보 (data.go.kr 15073875)
- **제공방식**: OpenAPI(JSON/XML), 실시간, 무료, 키필요, 제한없음
- **주요 필드**: **품목명, 모델명, 제조/수입사, 등급, 허가/인증/신고번호, UDI-DI(표준코드), 품목분류번호**, 멸균/일회용/인체이식 여부, 사용목적, 보관조건
- **포장단위**: 명시적 포장단위 필드 약함

- **평가(B)**: 모델명·표준코드(UDI-DI)·등급 등 식별 정보는 우수. 포장단위는 약해 직접 등록 보완 필요. **2차 후보.** (UDI-DI 보유 시 바코드 연계 강점)

- **키 없이 CSV로 즉시 확보 가능**: 심평원 약가마스터 의약품표준코드 (1차 import에 가장 마찰 적음)
- **OpenAPI만(키 필요)**: 묶음의약품, 의료기기, 기능성화장품, 식품류 대부분
- **식의약 데이터 포털(data.mfds.go.kr) / 의약품안전나라(nedrug.mfds.go.kr)** 는 동일 데이터를 CSV·EXCEL 파일로도 제공하는 경우가 있음 → 키 없는 일괄 확보 추가 경로 (개별 확인 필요, **E**)

## 12. 주요 키 필드 비교

| 상품군 | 대표상품 식별 후보 | 포장/SKU 식별 후보 | 바코드 연결 |
|---|---|---|---|
| 의약품 | 품목기준코드 / 묶음 대표 품목기준코드 | 표준코드(13자리) + 포장형태/총수량 | **표준코드→GS1 GTIN** |
| 의약외품 | 허가/신고번호 | (포장단위, 일관성 낮음) | 없음 |
| 건기식 | 품목제조신고번호 | (약함) | 없음 |
| 의료기기 | 허가/인증/신고번호 | UDI-DI | **UDI-DI(GS1)** |
| 화장품 | 보고번호/표준코드 | (없음) | 없음 |
| 일반식품 | 품목제조보고번호 | (약함) | 유통바코드(갱신중단) |

## 13. 대표상품 후보 생성 가능성

- **의약품: 가능(높음).** 묶음의약품 서비스의 "대표 제품" 개념이 O4O 대표상품과 직접 대응. 또는 약가마스터 대표코드/일반명코드로 그룹핑.
- **건기식/의료기기/의약외품: 부분 가능.** 제품 단위 레코드는 있으나 "대표 ↔ 변형" 묶음은 직접 구성 필요.
- **화장품/일반소비재: 어려움.** 브랜드/시리즈 그룹핑 정보 부족 → 직접 등록 권장.

## 14. 포장단위 후보 생성 가능성

- **의약품: 가능.** 약가마스터 `포장형태`+`제품총수량`+`약품규격`, 허가정보 `포장단위`. 가장 풍부.
- **나머지 상품군: 빈약.** 포장단위 필드가 없거나 비정형 → **빈 필드 허용 정책에 부합하게 비워두고 공급자 직접 등록**으로 처리하는 것이 현실적.

## 15. 기준 코드 후보

- **1순위 키 — 의약품 표준코드(13자리 KD코드)**: 고유·표준·바코드(GTIN) 연결 가능. SKU/포장단위 기준 키로 최적.
- **품목기준코드**: 대표상품/허가 단위 기준 키. (묶음 대표 품목기준코드 = 대표상품 키)
- **UDI-DI**: 의료기기 식별/바코드 키.
- 의약외품/건기식/식품 = 허가·신고·품목제조번호 (식별은 되나 바코드 연결 약함).

## 16. 1차 import 적합 후보 (A 등급)

1. **심평원 약가마스터 의약품표준코드 (15067462)** — 키 없는 CSV, 표준코드+포장형태+제조사+상품명 일괄. **최우선.**
2. **식약처 의약품 제품 허가정보 (15095677)** — 포장단위·전문일반·허가상태·품목기준코드. 약가마스터 보강.
3. **식약처 묶음의약품정보서비스 (15063908)** — 대표상품 그룹핑 시드.

→ **세 소스 결합 시 의약품에 한해 "대표상품 + 포장단위 + 기준코드 + 바코드 연결"의 후보 구조를 공공데이터만으로 구성 가능.**

## 17. 부적합·제외 데이터 (D 등급)

- **바코드연계제품정보 (15060549)** — 식품 한정 + 2018 갱신중단. 제외.
- 효능/용법/주의사항/DUR 등 임상·상담 필드 — 조사 목적상 수집/사용 제외.

## 18. 위험요소와 한계

- **인증키/호출제한**: OpenAPI는 개발 1만콜/일. 대량 일괄확보엔 운영키 신청 또는 CSV 경로 필요. (대량 다운로드는 본 IR 범위 밖 — 후속 WO에서 키·정책 확정)
- **이용허락 혼재**: "제한없음"(식약처)과 "출처표시 제1유형"(심평원 약가마스터) 혼재 → 출처표시 의무 준수 필요.
- **갱신주기 상이**: 의약품 허가정보=실시간 / 약가마스터=연1회 → 결합 시 시점 불일치 가능.
- **포장단위 비정형**: 의약품 외 상품군은 포장단위 표준화가 안 되어 직접 등록 의존.
- **개인정보**: 대상 데이터는 사업자/제품 정보로 개인정보 위험 낮음.

## 19. 결론

### 적합성 분류
- **A (1차 import)**: 약가마스터 표준코드(15067462), 의약품 제품 허가정보(15095677), 묶음의약품(15063908)
- **B (2차 후보, 정제필요)**: 의약외품(15095679), 건강기능식품(15056760), 의료기기 표준코드별(15073875)
- **C (참고)**: 기능성화장품(15095680), 식품 품목제조보고(15064909), 수입식품 DB(15073949)
- **D (제외)**: 바코드연계제품정보(15060549, 갱신중단)
- **E (추가확인)**: 식의약포털/의약품안전나라 CSV·EXCEL 일괄 다운로드 가능 범위 및 키 불필요 여부

### 상품군별 결론
- **의약품**: ✅ 공공데이터만으로 대표상품+포장단위+기준코드 후보 구성 가능. **최우선 import 대상.**
- **의약외품**: 🔶 제품 식별 가능, 포장/표준코드 약함 → 2차.
- **건강기능식품**: 🔶 제품·신고번호 확보, 포장단위는 직접 등록 보완.
- **의료기기**: 🔶 모델/UDI-DI 우수, 포장단위 약함 → 2차.
- **화장품**: ⚪ 기능성 한정·브랜드/포장 부재 → 직접 등록 권장.
- **일반 소비재**: ⚪ 커버리지·포장단위 빈약 → 직접 등록이 현실적, 공공데이터는 메타 보강용.

### 반드시 답할 질문 — 요약
1. 1차 후보: **심평원 약가마스터 표준코드 + 식약처 의약품 허가정보 + 묶음의약품(의약품 3종).**
2. 상품명+제조사+포장단위 모두 포함: **의약품 허가정보 / 약가마스터 = YES.** 그 외 상품군 = 부분적.
3. 기준 키: **의약품 표준코드(13자리), 품목기준코드, (의료기기)UDI-DI.**
4. 바코드 연결: **의약품 표준코드 → GS1 GTIN, 의료기기 UDI-DI.** 직접 바코드 데이터셋은 갱신중단(D).
5. CSV vs OpenAPI: **약가마스터 = 키 없는 CSV.** 그 외 대부분 = OpenAPI(키 필요).
6. 공공데이터만으로 구조 생성: **의약품만 가능.** 타 상품군은 포장단위 직접 등록 필요.
7. 직접 등록이 현실적인 군: **화장품, 일반 소비재(식품/생활/위생).** (건기식·의료기기는 포장단위 한정 직접 등록.)
