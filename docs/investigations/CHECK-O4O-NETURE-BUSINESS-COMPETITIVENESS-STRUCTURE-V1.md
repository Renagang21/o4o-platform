## 0. 핵심 결론 (TL;DR)

> O4O 경쟁력 구조는 다음 **5 축**으로 확정한다.
>
> **구매 / 상품 / 매장 / 운영·데이터 / 성장**
>
> IR §1.1–1.5 의 1차 가정 5 축 중 2 축의 호칭을 사업자 언어로 교체:
>
> - 공급 → **구매**
> - 자체 제품 → **상품**
> - 운영 → **운영·데이터** (데이터/AI 흡수)
>
> 페이지 구조는 **하이브리드 C** — 단일 개요 페이지(`/o4o/competitiveness`) 에 5 축 카드 노출 + 축별 상세는 후속 WO 에서 점진 확장.
>
> 각 축 카드의 설명 흐름은 **Pain → Gain → O4O 활용** 3-단 구조.

## 1. 경쟁력 축 재검토 — 결과

### 1.1 5 축 적절성 평가

| IR §1 축 | 사업자 직관성 | 평가 | 변경 결정 |
|---------|:-----------:|------|----------|
| 공급 경쟁력 | △ | "공급" 은 사업자 자신을 *받는 쪽* 으로 위치시키나, 능동성이 약함. 동일 의미를 사업자 시점에서 표현하면 "구매" | **구매 경쟁력** 으로 변경 |
| 자체 제품 경쟁력 | △ | "자체 제품" 은 PB 만 떠올리게 함. *차별화 상품 / 신제품 검증 / 공동 개발* 등 더 넓은 영역을 포괄하려면 "상품" | **상품 경쟁력** 으로 확장 |
| 매장 경쟁력 | ◯ | 직관적 / 명확 | **유지** |
| 운영 경쟁력 | △ | "운영" 단독은 추상. 데이터·AI 가 운영의 핵심 동력이 된 시대이므로 명시 권장 | **운영·데이터 경쟁력** 으로 통합 |
| 성장 경쟁력 | ◯ | 직관적 / 사업 확장·협동·지역 시너지 모두 포괄 | **유지** |

### 1.2 최종 5 축

| # | 축 | 한 줄 정의 |
|---|------|-----------|
| 1 | **구매 경쟁력** | 함께 사고, 더 좋은 조건으로 받습니다 |
| 2 | **상품 경쟁력** | 차별화된 상품 · 자체 상품을 만들 수 있습니다 |
| 3 | **매장 경쟁력** | 매장 자체가 더 강한 경험·설명 공간이 됩니다 |
| 4 | **운영·데이터 경쟁력** | 데이터와 AI로 더 똑똑하게 운영합니다 |
| 5 | **성장 경쟁력** | 혼자가 아닌 함께 성장합니다 |

→ 축 수 5 유지. 통합/삭제 없음.

## 2. 사업자 언어 변환 — 내부 도메인 용어 매핑

내부 도메인 용어를 사업자 언어로 1:1 매핑.

| 내부 용어 | 사업자 언어 (노출 후보) |
|---------|---------------------|
| Event Offer | 공동 구매 · 특가 행사 · 더 좋은 조건 구매 |
| Market Trial | 신제품 시장 검증 · 자체 상품 출시 가능성 · 시장 반응 보기 |
| Operator | 운영 파트너 · 공동 운영자 |
| Store Execution | 매장에서 실행 · 매장 행동으로 이어짐 |
| HUB | 공동 콘텐츠 · 공유 자산 |
| Service Scope | 사업 영역 · 사업 유형 |
| 무재고 판매 구조 | 재고 부담 없이 판매 · 위탁 판매 |
| Producer / Visibility | (개요 페이지 노출 안 함 — 내부 구조 용어) |
| Boundary / Tenant | (개요 페이지 노출 안 함) |
| RBAC | (개요 페이지 노출 안 함) |

**원칙**

- 내부 용어 단독 노출 금지
- 사업자 표현으로 변환 후, 신뢰 가교가 필요한 경우 *괄호 안에 내부 용어 병기* 가능 (예: "공동 구매 (Event Offer)")
- 본 페이지에서는 변환 표현만 사용, 병기는 상세 페이지에서 결정

## 3. O4O 기능 연결 — 5 축 × 기능 매트릭스

| 축 | O4O 기능 (주축) | O4O 기능 (보조) |
|----|----------------|----------------|
| 1. 구매 경쟁력 | Event Offer · 공동 구매 · 특가 행사 · 공급 통합 | HUB 상품 카탈로그 |
| 2. 상품 경쟁력 | Market Trial · PB / 자체 브랜드 · 공동 개발 | 콘텐츠 · 시장 데이터 |
| 3. 매장 경쟁력 | 콘텐츠 / QR / POP / 디지털 사이니지 / 키오스크·태블릿 / LMS / AI 설명 지원 | 무재고 판매 구조 |
| 4. 운영·데이터 경쟁력 | AI 활용 · 데이터 · 경험 공유 · 콘텐츠 자산화 · 운영 효율 | LMS · HUB |
| 5. 성장 경쟁력 | Operator · 협동조합 · 세미 프랜차이즈 · 사업 확장 · 지역 운영 | 공동 콘텐츠 · 공동 데이터 |

### 3.1 횡단 기능 (한 축에 묶이지 않음)

- **AI** — 매장·운영·상품·구매 4 축에 모두 걸침. 주축은 **운영·데이터** 로 배정 (가장 강한 가치 표현). 다른 축에서는 *보조* 또는 한 줄 언급.
- **콘텐츠** — 매장·운영·성장 3 축에 걸침. 주축은 **매장** (사업자가 가장 즉시 떠올리는 자리). *운영* 에서는 "콘텐츠 자산화" 로 다른 면 노출.
- **HUB** — 구매·매장·성장 3 축에 걸침. 직접 노출은 안 하고 *공동 콘텐츠 / 공동 자산* 등 표현으로 우회.

## 4. 경험 공유 위치 — 결정

### 4.1 후보 비교

| 위치 후보 | 강점 | 약점 |
|----------|------|------|
| 운영 경쟁력 안에 유지 | "다른 매장의 경험으로 내 운영을 개선" 흐름이 자연스러움 | 횡단 가치가 일부 가려짐 |
| 운영·데이터 경쟁력 (통합형) | 데이터·AI·경험 공유 3축이 한 묶음 — *학습형 경쟁력* 으로 응축 | "경험 공유" 가 데이터 보조처럼 보일 위험 |
| 별도 경쟁력 축 분리 | 가장 강조됨 | 6 축으로 늘어나 시각 부담 ↑, 다른 축과 의미 중복 |

### 4.2 결정

**운영·데이터 경쟁력 안에 통합 유지.** 단 매장·성장 축의 카피에 *경험 공유* 한 줄씩 노출하여 횡단 가치 보존.

근거:

- 5 축 구조 유지 (시각 부담 최소)
- "데이터로 운영 + 경험 공유 + AI" 가 한 묶음이 될 때 **학습형 경쟁력** 이라는 개념이 강해짐
- 매장(콘텐츠를 통한 경험 공유) 과 성장(공동 운영을 통한 경험 공유) 에서 자연스럽게 재등장 가능

## 5. 축별 사업자 공감 표현 — 후보

각 축 5~7개. 페이지 카드 본문 / 상세 페이지 핵심 진술의 소재.

### 5.1 구매 경쟁력

- 더 좋은 조건으로 구매할 수 있습니다
- 함께 구매해서 비용을 낮춥니다
- 단독으로 어려운 협상도 가능해집니다
- 안정적인 공급망을 확보합니다
- 좋은 상품을 더 빨리 만납니다
- 같은 업종 사업자들과 공동 대응합니다

### 5.2 상품 경쟁력

- 우리 매장만의 상품을 만들 수 있습니다
- 시장에 검증해보고 출시할 수 있습니다
- 차별화된 상품으로 경쟁합니다
- 자체 브랜드(PB) 가능성이 열립니다
- 제조사와 함께 상품을 만들 수 있습니다
- 새로운 상품을 작게 시도하고 확장합니다

### 5.3 매장 경쟁력

- 설명 없이도 고객이 이해합니다
- 매장이 정보 전달 공간이 됩니다
- 콘텐츠로 신뢰를 쌓습니다
- 고객 체류 시간과 재방문이 늘어납니다
- AI가 고객 응대를 돕습니다
- 다른 매장의 경험을 콘텐츠로 활용합니다

### 5.4 운영·데이터 경쟁력

- 데이터로 더 똑똑하게 운영합니다
- 경험을 자산으로 만듭니다
- AI가 반복 업무를 줄여줍니다
- 콘텐츠는 한 번 만들면 계속 쓰는 자산이 됩니다
- 다른 매장의 운영 노하우를 활용합니다
- 흩어진 정보를 모아 의사결정에 씁니다

### 5.5 성장 경쟁력

- 혼자가 아닌 함께 성장합니다
- 매장 그룹 · 협동조합 · 세미 프랜차이즈로 확장합니다
- 공동 운영 구조를 만듭니다
- 지역 안에서 시너지를 만듭니다
- 새로운 매장 형태를 작게 시도할 수 있습니다
- 사업 확장의 위험을 함께 나눕니다


### 6.3 `/o4o` 메인과의 위계

| 위치 | 역할 |
|------|------|
| `/o4o` (개요) | 왜 필요한가 + 무엇인가 + 누구를 위한 것 + 어떻게 작동 |
| `/o4o/competitiveness` | **무엇이 좋아지는가** (본 사이클) |

## 7. 경쟁력 설명 흐름 — 카드 내부 3-단

각 축 카드는 3 단으로 구성. 사업자 시점 → O4O 시점 순.

```text
[1] Pain (지금까지의 한계)
    "혼자서는 단가 협상이 어렵습니다"
   ↓
[2] Gain (좋아지는 것)
    "함께 구매해서 더 좋은 조건을 받습니다"
   ↓
[3] O4O 활용 (어떻게)
    "공동 구매 / 특가 행사 / 공급 통합"
```

본 흐름은 **5 축 모두 동일 구조** 로 적용 — 시각/리듬 일관성 확보.
