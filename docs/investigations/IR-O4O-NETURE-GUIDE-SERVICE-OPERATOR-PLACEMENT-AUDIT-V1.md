> **유형:** Read-only Investigation Report (조사 전용 — 코드/Guide/카드/구조 무수정)
> **작성:** 2026-06-07
> **선행:** [IR-O4O-NETURE-GUIDE-IA-AUDIT-V1](IR-O4O-NETURE-GUIDE-IA-AUDIT-V1.md) · Phase 1/2 (`49457fac8`·`431d9713a`) · Enrichment (`6418bca17`)
> **목적:** "O4O를 활용하는 **서비스 운영 사업자**" 개념을 현재 Guide IA(O4O 개요 / 공급자 / 운영자 / 판매자 / 파트너) 안에서 **어디에 위치시킬지** 판단. 신규 페이지 생성 여부가 아니라 **개념의 IA 배치**를 먼저 확정한다.

## 0. 요약 (TL;DR)

- **"서비스 운영 사업자"는 신규 역할이 아니다.** 철학 SSOT(O4O-BUSINESS-PHILOSOPHY-V1 §3.2)는 **운영사업자(Operator) = "서비스 운영 사업자"** 로 이미 정의한다. 즉 Guide의 `운영자` Lane이 곧 서비스 운영 사업자 Lane이다.
- **새로 드러난 것은 "운영자의 출신(origin)이 두 갈래"라는 점**이다 — **판매자 기반**(협동조합·약국 네트워크·전문매장 네트워크 등 매장들이 뭉쳐 운영자가 됨) vs **공급자 기반**(도매·제조·수입·총판·브랜드사가 매장 네트워크를 만들려 운영자가 됨). 이 구분은 4역할 분류를 **가로지른다(cross-cut)**.
- **현재 Guide는 판매자 기반 형태만 부분 노출**: `/guide/for-operator` #examples(협동조합·세미프랜차이즈·전문네트워크). **공급자 기반 운영자 개념은 Guide에 없음.**
- **"사업자 유형 누가·왜" 트랙은 이미 `/o4o/*`에 별도 존재**(`site-operator`=공급자 주도, `targets/*`=업종, `apply`=협동조합·협회 검토). Guide에 사업자 유형을 나열하면 이 Business 트랙과 **중복**된다.
- **권장 = 하이브리드 A+C (신규 Lane·신규 페이지 불필요).**
  - **A:** `/guide/o4o-overview`에 "O4O를 운영하는 서비스 운영 사업자 = 운영자이며, 판매자 기반·공급자 기반 두 출신이 있다"는 **개념 1섹션** 추가.
  - **C:** `/guide/for-operator` #examples를 **판매자 기반 / 공급자 기반 2그룹**으로 확장(형태 나열, 사업자 유형 상세는 `/o4o`로 링크).
  - **B(별도 페이지)·D(신규 Lane) 비권장** — 운영자 Lane·Business 트랙과 중복, 4역할 위 상위 계층 오해 유발.
- **진입 흐름:** `개요 → 4역할` 유지. 서비스 운영 사업자를 4역할 **위 상위 계층으로 두지 않는다**(그것은 운영자의 하위 framing이지 4역할의 부모가 아님).

## 1. 현재 Guide IA 상 위치 조사 (조사항목 1)

### 1-1. Guide(`/guide/*`) = 사용 안내(how) — role 기반

| 페이지 | 서비스 운영 사업자 개념 노출 정도 |
|--------|----------------------------------|
| `/guide` 허브 | 카드 "05 운영자 이용 안내" = 운영자 Lane 진입. "서비스 운영 사업자"라는 표현 자체는 없음 |
| `/guide/o4o-overview` | #roles에서 운영자 = "매장 네트워크를 구성·운영하는 사업자"로 1줄 정의. 출신 구분 없음 |
| `/guide/for-operator` | hero: 협동조합·협동조합 준비 그룹·세미프랜차이즈·전문네트워크 운영자 명시. #role: "참여 매장의 판매 환경을 구성하는 능동적 사업 주체"(Enrichment 반영). **#examples: 협동조합·세미프랜차이즈·전문네트워크 3형태** — 전부 **판매자 기반** |
| `/guide/for-seller` | 판매자(매장) 관점 — 운영자 네트워크에 "참여하는" 입장. 운영 주체 설명 아님 |

> **결론:** "서비스 운영 사업자" 개념은 현재 **운영자 Lane(특히 for-operator hero·#examples)에서만**, 그것도 **판매자 기반 형태로만** 다뤄진다. O4O 개요에는 개념 정의 1줄뿐, 공급자 기반 운영자는 어디에도 없다.

### 1-2. 인접 `/o4o/*` = 사업 소개·랜딩(who·why) — Guide 밖에 이미 존재

| 라우트 | 역할 | 서비스 운영 사업자 관련 |
|--------|------|------------------------|
| `/o4o` (O4OMainPage) | 플랫폼 개념·운영 원칙 | 4대 운영 원칙(매장 실행 중심 등) |
| `/o4o/site-operator` | **기존 사이트 운영 사업자 대상** | **공급자·본부·마케팅 파트너** = 사실상 **공급자 기반 운영자** 랜딩 |
| `/o4o/targets/{pharmacy\|clinic\|salon\|optical\|dental}` | 업종별 매장 모집 카피 | 매장 기반 사업 그룹 대상 |
| `/o4o/apply` (O4OApplyPage) | 적용 검토 신청 | **협동조합·협회·전문가 단체**를 검토 대상으로 명시 |

> **결론:** "**어떤 사업자가 O4O를 쓰는가(사업자 유형·니드·랜딩)**"는 **이미 `/o4o/*` Business 트랙**이 소유한다. 선행 [IR-O4O-NETURE-BUSINESS-TARGET-STRUCTURE-V1](IR-O4O-NETURE-BUSINESS-TARGET-STRUCTURE-V1.md)이 5 사업자 유형(연합체/매장그룹/공급망/전문서비스/지역운영)을 구조화하는 트랙(개요→경쟁력→**사업자 유형**→사례→Landing)이 별도로 돌고 있다. Guide는 그 트랙의 "사용 안내" 짝이다.

## 2. 개념 정리 (조사항목 2 — 중복 여부)

### 2-1. 역할(Role) vs 서비스 운영 사업자(Business Entity)

| 축 | 정의 | 값 |
|----|------|----|
| **역할(Role)** | 거래·실행 흐름에서 수행하는 **기능** | 공급자 · 운영자 · 판매자 · 파트너 |
| **서비스 운영 사업자** | O4O 서비스를 **운영하기로 한 사업 실체** | = **운영자 역할**을 맡은 실제 조직 |

- 철학 §3.2: **운영사업자(Operator) = "서비스 운영 사업자"** → 둘은 **같은 것의 두 표현**(역할 이름 vs 사업 실체 이름)이다. **신규 개념·신규 역할이 아니다.**
- 따라서 Guide에 "서비스 운영 사업자"를 **운영자와 별개 페이지**로 만들면 → **운영자 Guide와 정의가 충돌·중복**된다. (이용자: "운영자랑 서비스 운영 사업자랑 뭐가 다르지?")

### 2-2. 진짜 새로운 축 — 운영자의 "출신(origin)"

서비스 운영 사업자는 **어느 역할에서 출발했는가**로 두 갈래다. 이 축이 4역할을 **가로지른다**:

| 출신 | 사업자 예 | 현재 Guide 노출 |
|------|-----------|----------------|
| **판매자 기반** (매장들이 뭉쳐 운영자가 됨) | 협동조합 · 관광객 대상 약국 네트워크 · 창고형 약국 네트워크/협의체 · 화장품 전문매장 네트워크 · 전문약사 네트워크 · 특정 질환 전문 네트워크 · 지역 약국 네트워크 | △ for-operator #examples(협동조합·세미프랜차이즈·전문네트워크) |
| **공급자 기반** (공급자가 매장 네트워크를 만들려 운영자가 됨) | 도매상 주도 · 제조사 주도 · 수입사 주도 · 총판 · 브랜드사 | ❌ Guide 없음 (`/o4o/site-operator`에만 랜딩 카피) |

> **이것이 본 IR의 핵심 발견.** "협동조합을 어디 넣을까"가 아니라, **운영자가 판매자 출신일 수도, 공급자 출신일 수도 있다**는 사실을 Guide가 아직 표현하지 못한다는 점이다. 공급자 기반 운영자(도매·제조·수입)는 Guide 전체에서 **완전 공백**이다.

### 2-3. 중복 위험 요약

| 만약 ~로 배치하면 | 중복/충돌 대상 |
|------------------|----------------|
| 운영자와 **별도 역할/Lane** | 운영자 Lane (정의가 동일 → 직접 충돌) |
| 사업자 유형을 **Guide에서 나열·설명** | `/o4o` Business 트랙(사업자 유형·니드·랜딩) |
| 4역할 **위 상위 계층** | 4역할 분류 체계(운영자는 4역할 중 하나이지 4역할의 부모가 아님) |

## 5. Guide 진입 흐름 조사 (조사항목 5)

**현재:** `O4O 개요 → 공급자 / 운영자 / 판매자 / 파트너` (4역할 병렬)

**검토안:** `O4O 개요 → 서비스 운영 사업자 → 공급자 / 운영자 / 판매자 / 파트너`

→ **부적절.** 서비스 운영 사업자는 운영자 역할의 **사업 실체 framing**이지, 4역할 **전체의 부모가 아니다**. 공급자·판매자·파트너가 "서비스 운영 사업자"의 하위가 되는 것은 분류 오류다.

**권장 흐름(유지):**
```
O4O 개요
  └ (개요 안에서) 서비스 운영 사업자 = 운영자, 판매자 기반·공급자 기반 두 출신 — 개념만
  ↓
공급자 / 운영자 / 판매자 / 파트너   ← 4역할 병렬 유지
        └ 운영자 Lane 안에서 판매자 기반/공급자 기반 형태 상세
```
즉 **개념은 개요(상위)에서 1회, 형태는 운영자 Lane(하위)에서 전개**. 4역할 병렬 구조는 그대로.

## 6. 권장안

> **하이브리드 A + C. 신규 페이지·신규 Lane·허브 카드 추가 없음.**

1. **(A) `/guide/o4o-overview`** — #roles 다음 또는 #actors 근처에 **개념 1섹션** 추가:
   - "O4O를 운영하는 주체 = 서비스 운영 사업자 = 운영자"
   - 운영자는 **판매자 기반**(매장 연합·협동조합·전문매장 네트워크)일 수도, **공급자 기반**(도매·제조·수입·총판·브랜드사)일 수도 있다 — **개념과 두 출신만**.
   - 사업자 유형 상세·랜딩은 `/o4o`로 안내(링크 문구).
2. **(C) `/guide/for-operator` #examples 확장** — 현재 3형태(협동조합·세미프랜차이즈·전문네트워크)를 **판매자 기반 / 공급자 기반 2그룹**으로 재구성. **공급자 기반(도매·제조·수입) 공백을 채움**. 각 항목은 형태 설명만(특정 조직·성공사례 금지 — Enrichment 원칙 유지).
3. **B·D 폐기.** 별도 페이지/Lane은 운영자 Lane·`/o4o` Business 트랙과 중복.
4. **`/o4o` Business 트랙과의 관계 명시** — Guide(사용 안내)는 형태·개념까지, 사업자 유형 누가·왜·랜딩은 `/o4o`. **Guide에서 사업자 유형을 재서술하지 않는다**(중복 0).

### 권장 근거 한 줄
> 운영자 = 서비스 운영 사업자(철학 §3.2)이므로 **별도 개념이 아니라 운영자 Lane의 보강**이고, 새로 채울 진짜 공백은 **"공급자도 운영자가 될 수 있다"(공급자 기반 운영자)** 한 가지다.
