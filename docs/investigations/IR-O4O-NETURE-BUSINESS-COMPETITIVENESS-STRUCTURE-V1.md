## 0. 조사 목적

사업자가:

> "내 사업에서 무엇이 좋아지는가"

를 빠르게 이해하도록 만든다.

본 조사의 핵심은 **철학 설명이 아니라 경쟁력 구조 설명**이다.

### 본 IR 의 큰 그림 안의 위치

```text
개요         = 왜 필요한가     (완료 — WO 인계)
경쟁력 구조  = 무엇이 좋아지는가  ← 본 IR
사업자 유형  = 누가 쓰는가      (예정)
사례         = 어떻게 쓰는가     (예정)
```

## 2. 조사 항목

### 2.1 경쟁력 분류 조사

**조사**

- 5 축이 적절한가
- 빠진 것이 있는가
- 너무 복잡한가
- 통합 가능한 축이 있는가

**정리**

- 최종 축 수 (3~5 권장)
- 통합 / 분리 결정
- 누락 영역 보강

---

### 2.2 사업자가 공감하는 표현 조사

**좋은 표현 후보**

- 더 저렴하게 구매
- 더 강한 설명 경쟁력
- 자체 제품 가능성
- AI 활용
- 운영 효율
- 매출 향상
- 고객 재방문
- 객단가 상승

**피해야 할 표현**

- 플랫폼 기능 설명
- "Event Offer / Market Trial" 등 내부 도메인 용어 단독 노출
- 추상 가치 진술 ("혁신적인 / 차별화된 / 강력한")

**정리**

- 각 축마다 사업자 표현 후보 정리
- 내부 용어 → 사업자 언어 변환 매핑

---

### 2.3 O4O 기능 연결 조사

각 경쟁력이 O4O 구조와 어떻게 연결되는가 매핑.

**예시 (1차 가정)**

```text
Event Offer     → 공급 경쟁력
Market Trial    → 자체 제품 경쟁력
콘텐츠/QR/POP   → 매장 경쟁력
데이터/AI       → 운영 경쟁력
Operator/협동조합 → 성장 경쟁력
```

**정리**

- 각 축 → O4O 기능 매핑
- 한 기능이 여러 축에 걸쳐 있을 경우 주축 결정

---

### 2.4 설명 구조 조사

각 경쟁력 영역을 어떤 흐름으로 설명할 것인가.

**후보 흐름**

```text
경쟁력 (사업 관점)
   ↓
어떻게 만드는가 (방법 / 메커니즘)
   ↓
O4O 활용 (실제 기능)
```

또는

```text
사업자 문제 (Pain)
   ↓
경쟁력 (Gain)
   ↓
O4O 활용
```

**정리**

- 단일 흐름 채택 vs 축별 흐름 다양화
- Hero / Card / Detail 단계별 메시지 깊이

---

### 2.5 페이지 구조 조사

본 경쟁력 설명을 어떤 페이지/섹션 단위로 노출할 것인가.

**후보**

- A. **단일 페이지** — `/o4o/competitiveness` 1 페이지 안에 5 축 모두 노출
- B. **축별 분리** — `/o4o/competitiveness/supply`, `/store`, `/operation` 등 5 페이지
- C. **하이브리드** — 개요 페이지(5 축 카드) + 각 축 상세 페이지

**정리**

- 페이지 수 결정
- 진입 동선 결정
- 본 영역과 `/o4o` 메인의 관계 (§3.7 활용 방식 / §3.8 대표 기능과의 위계)

---

### 2.6 사업자 유형 연결 조사

각 경쟁력 축이 어떤 사업자 유형에게 가장 강하게 다가가는가.

**예 (1차 가정)**

- 공급 경쟁력 → 약국 협동조합 / 매장 그룹 / 의료기관
- 자체 제품 경쟁력 → 제조사 / 매장 그룹 / 화장품 매장
- 매장 경쟁력 → 모든 매장
- 운영 경쟁력 → 다점포 / 매장 그룹 / 의료기관
- 성장 경쟁력 → 협동조합 / 투자자 그룹 / 세미 프랜차이즈

**정리**

- 축 × 사업자 유형 매트릭스 초안 (다음 사이클 입력)
