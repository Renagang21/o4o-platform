## 1. 참고 법령 / 공식 자료 요약

### 1.1 전자상거래법 제10조 — 사이버몰 운영자 초기화면 표시사항 (필수)

전자상거래를 하는 사이버몰 운영자는 소비자가 사업자 신원을 쉽게 알 수 있도록 **초기 화면**에 다음을 표시해야 한다 (총리령 기준):

1. 상호 및 대표자 성명
2. 영업소가 있는 곳의 주소 (소비자 불만 처리 가능한 주소 포함)
3. 전화번호
4. 전자우편주소
5. 사업자등록번호
6. 사이버몰의 이용약관 (소비자가 **연결 화면**을 통해 볼 수 있도록 가능)
7. 호스팅서비스를 제공하는 사업자의 상호

추가: 표시사항의 진위를 소비자가 확인할 수 있도록 **공정거래위원회가 운영하는 사업자정보 공개페이지를 초기 화면에 연결**해야 한다.

### 1.2 통신판매업 신고 (전자상거래법 §12)

통신판매업자는 통신판매업 신고를 하고 **통신판매업 신고번호**를 표시·고지해야 한다 (일정 거래규모·면제 기준 존재 — 서비스별 신고 의무 여부 확인 필요).

### 1.3 통신판매중개자의 의무·책임 (전자상거래법 §20, §20-2)

- §20① — 통신판매중개자는 **자신이 통신판매의 당사자가 아니라는 사실**을 소비자가 쉽게 알 수 있도록 미리 고지해야 한다.
- §20② — 통신판매중개업자는 중개의뢰자(판매자)가 사업자인 경우 그 성명·주소·전화번호 등을 확인하여 **청약 전까지 소비자에게 제공**해야 한다.
- §20-2 — 위 고지를 하지 않은 경우 중개의뢰자의 고의·과실로 소비자에게 발생한 손해를 **연대 배상**. 또한 통신판매업자인 중개자는 §12~§15, §17, §18 책임을 면하지 못함.

### 1.4 실무 권장 (쇼핑몰 초기화면 통상 표시)

개인정보처리방침 링크, 개인정보보호책임자, 통신판매업 신고번호, 호스팅 제공자, 사업자정보확인/공정위 통신판매사업자 조회 링크, (선불·배송상품 시) 구매안전서비스·에스크로 표시 등.

> 출처는 §13 참고자료 참조.

## 9. 후속 WO에서 구현 가능한 항목 (정보 확정 전제)

1. 공통 `Footer` 컴포넌트 추출(`packages/ui` 또는 신규) + 서비스별 설정(brand/연락처/사업자정보) 주입 구조 — 하드코딩 → 중앙관리.
2. 전상법 §10 7항목 + 호스팅 + 공정위 링크 슬롯 표준화.
3. 약관/개인정보처리방침 route 정비: KPA orphaned 페이지 연결 / GP·KCos route 신설(또는 Neture식 CMS LegalPage 패턴 cross-service 도입).
4. 통신판매중개자 고지 슬롯(중개 성격 확정 서비스 한정).
5. 푸터 정보 소스: env 또는 CMS(Neture LegalPage 패턴) 통일.

## 10. 후속으로 사용자 확인이 필요한 항목

§7 표의 11개 항목 전부 + "서비스별 거래 성격(판매/중개)" 확정. 특히 **운영 주체 단일/분리**와 **각 서비스 사업자등록번호·통신판매업 신고번호 실값**은 사용자 제공 없이는 구현 불가.

## 11. 푸터 구현 권고안 (정보 확정 후)

1. **단계 0 (선행)**: 운영 주체·법정정보 실값 확정 (사용자) + 거래 성격 법무 확인.
2. **단계 1**: 공통 Footer 컴포넌트 + 서비스 설정 주입. 전상법 §10 슬롯 표준화. placeholder 전면 제거.
3. **단계 2**: 약관/개인정보처리방침 route cross-service 정비(CMS LegalPage 패턴 우선).
4. **단계 3**: 통신판매중개 고지 + 공정위 사업자정보 공개페이지 링크 + (해당 시) 구매안전서비스 표시.
5. **단계 4**: 안내 진입점(서비스 안내/이용 안내) + 문의하기 + 약관 + 개인정보 + 사업자정보를 푸터 IA로 통합.
