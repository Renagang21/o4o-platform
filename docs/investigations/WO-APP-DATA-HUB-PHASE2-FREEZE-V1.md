## 0. 목적 (Purpose)

APP Data Hub의 **Phase 2 (2-A + 2-B)** 결과를
하나의 **완결된 사용자 경험 기준선**으로 고정한다.

본 문서 이후의 변경은 **Phase 3 (AI/자동화)** 또는
별도 확장 단계로만 진행한다.

---

## 1. Phase 2 기준선 선언 (Declaration)

**APP Data Hub – Phase 2는 다음 상태로 고정된다.**

* 허브 → 승인 → 즉시 이용 가능 흐름 완성
* 허브 자산 복사 시 **사용자 선택 UX 도입**
* 정책(OPEN / REQUEST_REQUIRED / DISPLAY_ONLY / LIMITED) 시각화 완료
* 신규 엔티티 없이 확장 가능한 구조 확보

👉 이 시점 이후:

* 복사 UX의 기본 흐름 변경 ❌
* 승인 정책 구조 변경 ❌
* Phase 2 범위 내 기능 추가 ❌
  (단, 버그 수정은 허용)

## 3. 확정된 사용자 흐름 (고정)

```
허브 카드
 → [내 대시보드로 복사]
   → 옵션 선택 모달
     (시작 방식 / 노출 방식)
       → 내 대시보드 생성
         → 메인 페이지 즉시 반영
```

* "복사 = 즉시 생성" 모델은 Phase 2 종료
* "복사 = 시작 방식 선택"이 기본 UX로 고정

---

## 4. UX 기준선 (고정)

### 4-1. 복사 UX 원칙

* ❌ 설정 화면처럼 보이지 않을 것
* ❌ 옵션 과다 노출 금지
* ✅ 시작 방식 선택이라는 인식
* ✅ 10초 이내 결정 가능

### 4-2. LIMITED 조건 표현 원칙

* 숨김 ❌
* 차단 ❌
* **설명 + 조건 명시**만 제공
* 사용자의 판단을 존중
