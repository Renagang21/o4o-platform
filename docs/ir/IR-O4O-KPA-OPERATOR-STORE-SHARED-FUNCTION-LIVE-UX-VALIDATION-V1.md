## 4. 실측으로 확정된 불일치 (정적 조사 보강)

### 4.1 상태 라벨 — **3종 분기 (실측, 최우선 정비)**
| 영역 | draft | published | archived |
|---|---|---|---|
| 운영자 blog/pop/qr/video | **초안** | **발행** | **보관** |
| 운영자 다국어 콘텐츠 | 초안 | **발행됨** | **보관됨** |
| 내 매장 블로그 | **임시저장** | **발행됨** | 보관 |

→ 동일 개념 상태가 **초안/임시저장**, **발행/발행됨**, **보관/보관됨** 으로 한 서비스 안에서 3 변형. 정적 조사(§7.2)의 "상태 어휘 2종"보다 실제로 더 분산. **단, 이는 라벨(표시) 통일이며 상태 값/발행 의미 변경(PARITY §R6, 고위험)과 구분** — 라벨만 통일하면 저위험.

### 4.4 "콘텐츠 허브" 명칭 충돌 (실측 확정)
- 운영자 `/operator/docs` 페이지 제목 = **"콘텐츠 허브"** (`kpa_contents`)
- 매장 `/store-hub/content` 내 탭 = **"콘텐츠 허브"** (운영자 게시물 선반)
→ 서로 다른 두 개념이 같은 이름. 정적 조사 §6 예측 그대로 실측됨. 명칭 통일 시 **혼동 위험**(D 분류) — 둘 중 하나 개명 필요.

### 4.5 빈화면 문구 분기 (실측)
"아직 **작성한** POP 이 없습니다"(운영자) / "아직 **운영자 게시** POP 이 없습니다"(HUB) / "아직 **만든** POP이 없습니다"(내매장) / "게시글이 없습니다"(블로그) / "등록된 자체 상품이 없습니다". 작성/등록/게시/만든 혼용.

### 4.6 ⚠️ [정정됨] "Not the store owner" = 권한 **결함** (보호 게이트 아님)

> **정정(2026-06-27):** 본 항목 최초 작성 시 "소유 가드 정상 동작"으로 판단했으나 **오판이었다.** O4O 정책상 매장 경영자가 서비스 가입을 신청하고 운영자가 승인하면 별도 소유자 지정 없이 즉시 내 매장을 이용할 수 있다. 따라서 **승인된 약국 경영자(`renagang21`)에게 `Not the store owner`(403)가 뜨는 것은 정상 차단이 아니라 결함**이다.

근본 원인(read-only 확정): o4o-store `blog/pop/qr/video` staff 컨트롤러가 소유권을 `created_by_user_id`(생성자)로 검사 — RBAC SSOT(`role_assignments.kpa:store_owner`)가 아님. `renagang21` 은 `kpa:store_owner` role 보유 + org `c92b857f`(테스트 약국, `organizationRole=owner`)이지만 약국 레코드 생성자가 아니라 403.

→ 수정: **`WO-O4O-KPA-APPROVED-STORE-OWNER-AUTO-AUTHORIZATION-FIX-V1`** (commit `d6134eea1`). `verifyOwner`를 `isStoreOwner`(role) + org 매칭으로 교체(store-content 선례 동일). 영어 메시지→한글. PARITY §R2 의 "created_by vs role" 도 "결함"으로 재해석. **교차 매장 차단은 유지**(resolved org === store.id).

### 4.7 콘텐츠 출처 탭 실측 (copy 모델 가시화)

## 5. ListColumnDef 수렴의 사용자 체감 (실측 판단)

정적 조사 §8 의 "내 매장 자체 테이블만 `@o4o/ui Column`" 비대칭은 **내부 구현 차이**다. 실측 결과:
- 운영자/HUB 리스트(ListColumnDef): 상태 탭(전체/초안/발행/보관) + 컬럼(액션/제목/슬러그/상태/수정일/발행일) 존재.
- 내 매장 리스트(@o4o/ui): 블로그도 상태 탭(전체/임시저장/발행됨/보관) + 컬럼(제목/슬러그/상태/수정일/액션) 존재.

→ **두 스택 모두 탭·정렬·행 액션을 제공** → 사용자 체감 차이는 작다. 테이블 API 수렴은 **유지보수상 이점**은 있으나 **이번 체감 개선 우선순위에서는 후순위**. (사용자 요청대로 운영 화면 확인 후 판단 = **후순위 확정**.)
