# O4O Platform Constitution v1

> O4O 플랫폼의 5대 영역별 책임과 한계를 정의하는 최종 기준선.
> 설계·구현 지침이 아닌 **판단 기준 문서**이다.

*Created: 2026-01-29 | Version: 1.0 | Status: Active Constitution*

---

## 계층 구조

```
Product DB (불변 사실)
    ↓ 소비
Content (표현·전달)
    ↓ 조합
Service (흐름·경험)
    ↓ 적용
Operation (유통 질서·운영 규칙)
    ↓ 보조
AI (반복 감소·요약·판단 보조)
```

각 계층은 하위 계층의 책임을 침범하지 않는다.

---

## 1. Product DB — 불변 사실의 저장소

> **상품 DB는 불변 사실(Immutable Fact)만을 담는다.**
> 설명, 표현, 해석, 마케팅 문구는 콘텐츠 또는 서비스의 책임이다.

### DB 승격 5원칙

새 필드를 상품 DB에 추가하려면 **모두 충족**해야 한다:

1. **서비스 불변성** — 어떤 서비스에서도 동일한 정보
2. **사실성** — 객관적 사실 (설명·해석 아님)
3. **공급자 원본 책임** — 생성·변경 책임이 공급자에게 있음
4. **콘텐츠 독립성** — 콘텐츠가 변해도 원본 값은 불변
5. **유통 질서 필수성** — 이 정보 없이 유통 판단 불가

### 승격 가능 항목

| 분류 | 예시 |
|------|------|
| 법적·제도적 사실 | 생산국, 제조사, 법적 분류 |
| 인증·허가 식별자 | 허가번호, 코드, ID |
| 유통 판단 조건 | 사업자 유형별 취급 가능 여부 |
| 불변 식별자 | SKU (내부), 바코드 (외부) |

### 명시적 제외

설명 텍스트, 마케팅 문구, 이미지/동영상, 추천·연관 정보, 노출·정렬 속성 → **콘텐츠/서비스 책임**

### SKU / Barcode 정책

- SKU = 내부 불변 식별자 (대체 불가)
- Barcode = 외부 유통용 (SKU에 연결되는 속성)
- SKU를 바코드로 대체 금지, 바코드 없이도 등록 가능

## 2. Content — 표현과 전달

> **콘텐츠는 표현과 전달을 담당한다. 사실을 정의하지 않고, 유통 질서를 결정하지 않는다.**

### 핵심 원칙

1. 상품 DB의 불변 사실을 **변경하지 않는다**
2. 서비스별로 **다르게 구성**될 수 있다
3. 표현 자산(텍스트, 이미지, 동영상, PDF)을 **관리**한다
4. 재편집·재배치·요약 **가능** (고정되지 않음)
5. 판단의 근거가 **아니다** (참고 자료일 뿐)

### 명시적 제외

상품 법적 사실 정의, 유통 판단, 가격/승인 결정, SKU/바코드 관리, 운영 규칙 대체

---

## 3. Service — 흐름과 경험

> **서비스는 흐름과 경험을 제공한다. 상품과 규칙을 정의하지 않는다.**

### 핵심 원칙

1. 상품 DB를 **읽기 전용**으로 사용
2. 콘텐츠를 **조합·배치·전달** (소유하지 않음)
3. 서비스별 UX 차이는 **허용**
4. 유통 질서를 **해석하지 않음**
5. 특정 서비스만의 예외 규칙 **금지**

### 명시적 제외

상품 DB 구조 변경, 유통 판단 내재화, 운영 규칙 하드코딩, 콘텐츠를 사실로 승격

---

## 4. Operation & Distribution — 유통 질서

> **유통 질서는 운영 규칙으로 표현된다. 상품과 서비스는 이를 소유하지 않는다.**

### 핵심 원칙

1. 공급자 → 판매자 **승인·제어** (운영 규칙)
2. 판매자별 **상품 노출 차등** 허용
3. **가격 등급** = 운영 규칙 (일반/우수/특별)
4. **기간·조건** = 운영 규칙 (공동구매, Trial, 수량 제한)
5. 반품·회수 = **공급자-판매자 간 계약** (플랫폼 책임 아님)

### 명시적 제외

상품 DB에 운영 로직 삽입, 서비스 화면에 규칙 하드코딩, 콘텐츠로 규칙 대체

---

## 5. AI — 보조 도구

> **AI는 판단을 돕는다. 책임을 지지 않는다.**

### 핵심 원칙

1. 상품 DB를 **수정하지 않는다**
2. **반복 작업 감소**에만 사용 (요약, 정리, 보조 설명)
3. **리뷰·보조 콘텐츠**에 한정
4. 운영·유통 **판단의 주체가 아니다** (가격, 권한, 승인 결정 금지)
5. 기존 UX 위에 **보조적으로** 동작

### 명시적 제외

상품 구조 자동 생성, 가격·권한·유통 판단, 법적 책임 판단, 정책 결정

---

## 변경 조건 (공통)

각 영역의 헌법은 다음 경우에만 재검토된다:

- 해당 영역에서만 해결 가능한 **법적·규제 요구** 발생
- 상위 헌법(Product DB)이 변경되어 **근거가 달라진** 경우
- 해당 영역이 다른 영역을 **침범하고 있다는 명확한 사례** 발생

그 외의 경우, 본 헌법은 변경되지 않는다.
