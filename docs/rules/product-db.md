# Product DB Constitution v1

> **O4O 플랫폼 상품 데이터베이스 기준 헌법**

---

## 문서의 성격

본 문서는 O4O 플랫폼에서 **상품 데이터베이스에 무엇을 담을 수 있는지**를 판단하는 최종 기준선이다.

* 이 문서는 설계 문서가 아니다
* 이 문서는 구현 지침이 아니다
* 이 문서는 **판단 기준 문서**이다

**적용 대상**:
- 내부 기획자
- 개발자
- 외부 AI 에이전트
- 협업자

---

## 핵심 원칙

### 상품은 데이터베이스이며 서비스가 아니다

상품 DB는 **불변 사실(Immutable Fact)**만을 담는다.
설명, 표현, 해석, 마케팅 문구는 콘텐츠 또는 서비스 영역의 책임이다.

---

# DB 구조 승격 5원칙 (Immutable Fact Rule)

상품 DB에 새로운 필드(구조)를 추가하려면,
아래 **5가지 기준을 모두 충족**해야 한다.

1. **서비스 불변성**

   * 이 정보는 어떤 서비스(O4O, Neture, K-Cosmetics, 사이니지 등)에서도 동일해야 한다.

2. **사실성(Fact)**

   * 이 정보는 설명·표현·해석이 아닌 객관적 사실이다.

3. **공급자 원본 책임**

   * 이 정보의 생성·변경·오류에 대한 1차 책임은 공급자에게 있다.

4. **콘텐츠 독립성**

   * 콘텐츠가 가공·편집되더라도 이 정보의 원본 값은 변하면 안 된다.

5. **유통 질서 필수성**

   * 이 정보가 없으면 유통 질서(판매 가능 여부, 규제 판단, 책임 추적)를 표현할 수 없다.

> 위 기준 중 하나라도 충족하지 못하는 정보는
> **상품 DB가 아니라 콘텐츠/서비스/운영 영역에 둔다.**

---

# DB 승격 확정 리스트 v1

*(Product Immutable Fact Layer)*

다음 항목만이 **상품 DB 구조로 승격 가능**한 정보이다.

## 1. 법적·제도적 사실 정보

* 생산국 / 원산지
* 제조사 / 수입사
* 상품의 법적 분류
  (예: 일반상품, 건강기능식품, 의료기기, 의약외품 등)

## 2. 인증·허가의 존재 사실 (식별자 수준)

* 인증/허가 번호, 코드, ID
  (예: 건강기능식품 허가번호, 의료기기 허가번호)

※ 제외:

* 인증 설명 문구
* 홍보용 이미지
* 인증 상세 콘텐츠

## 3. 유통 질서 판단에 필요한 고정 조건

* 특정 사업자 유형만 취급 가능 여부
* 특정 채널/조건에서의 판매 가능 여부

※ "어떻게 노출할지"는 서비스 영역,
※ "가능/불가능 여부"만 상품의 사실로 취급

## 4. 상품 식별의 불변 기준

* 내부 상품 식별자 (SKU)
* 외부 연동 식별 코드 (제조사 코드, 외부 시스템 코드 등)

---

# SKU / Barcode 식별 체계 (확정 정책)

## 1. SKU (내부 식별자)

* 플랫폼 내부의 **불변 식별자**
* 유통, 정산, 이력, 권한 판단의 기준
* 서비스/채널/표현과 무관
* **대체되지 않는다**

## 2. Barcode (외부 식별자)

* 외부 유통·현장·POS·스캔을 위한 표준 식별자
* 제조사/공급자 책임
* **SKU에 연결되는 속성**
* 상품 또는 옵션 단위로 연결 가능

## 3. 명시적 금지

* SKU를 바코드로 대체하지 않는다
* 바코드 변경 시 SKU를 재발급하지 않는다
* 바코드를 시스템 기준 ID로 사용하지 않는다
* 바코드가 없다고 상품 등록을 제한하지 않는다

---

## 명시적 제외 항목 (DB 책임 아님)

다음 항목은 의도적으로 상품 DB에서 제외한다.

* 상세 설명 텍스트
* 요약 문구 / 마케팅 문장
* 이미지 / 동영상 / 설명서 파일
* 추천·연관·조합 정보
* 서비스별 노출·정렬·표현 속성

→ **콘텐츠 또는 서비스의 책임 영역**

## 변경 조건

본 기준을 변경하려면 다음 조건을 충족해야 한다:

1. 변경 사유가 명확히 문서화되어야 함
2. 기존 구조와의 충돌 분석이 완료되어야 함
3. 변경 사항이 CLAUDE.md 규칙에 따라 승인되어야 함
