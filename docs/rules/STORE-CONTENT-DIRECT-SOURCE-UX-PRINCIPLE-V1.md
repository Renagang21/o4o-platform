# 공용 자료 직접 불러오기 기반의 매장 콘텐츠 작업 공통 UX 원칙

> **문서 ID:** STORE-CONTENT-DIRECT-SOURCE-UX-PRINCIPLE-V1
> **상태:** Active
> **작성일:** 2026-04-20
> **적용 범위:** QR-code, POP, 디지털사이니지, 블로그 글, 타블렛/전자상거래용 상품 정보 제작 및 기타 매장 콘텐츠 작업 기능 전반

---

## 1. 문제 정의

매장에서 콘텐츠를 작업할 때 현재 일부 기능은 다음 흐름을 따른다.

```
공용 자료 탐색 → 내 매장으로 가져오기 → 내 매장에서 선택 → 작업 실행
```

이 흐름은 "공용 자료를 일단 내 매장 자산으로 복사하는 것"을 작업의 전제 조건으로 삼는다.

---

## 2. 기존 방식의 한계

| 문제 | 설명 |
|------|------|
| 불필요한 선행 단계 | 자료 보관 자체보다 작업 실행이 목적인 경우, 복사 단계는 마찰이다 |
| 매장 자산 오염 | 실제로 저장할 의사 없이 탐색·비교·시험 목적으로 가져온 자료가 내 매장에 누적된다 |
| 자료 신선도 단절 | 복사본은 공용 자료 원본이 업데이트되어도 자동 반영되지 않는다 |
| 흐름 단절 | 사용자는 "자료 탐색"과 "작업 실행" 사이에 맥락을 잃는다 |

---

## 3. 새로운 UX 원칙

### 원칙 1 — 작업 화면에서 직접 불러오기를 기본으로 한다

공용 자료(자료실, 디지털사이니지 소스, 제조사 제품 정보 등)는 **작업이 필요한 화면에서 직접 참조하여 사용**한다.  
"내 매장에 먼저 가져오기"는 선행 조건이 아니다.

### 원칙 2 — 내 매장 저장은 선행 조건이 아니라 결과 단계의 선택이다

작업을 마친 결과물을 반복 사용하거나 매장 전용 편집본으로 보관할 필요가 있을 때만, 사용자가 **명시적으로** 내 매장 자산으로 저장한다.

### 원칙 3 — 공용 자료 참조와 매장 자산 저장을 분리한다

```
공용 자료 참조 (읽기 전용 연결)  ≠  내 매장 자산 저장 (소유권 이전)
```

두 행위는 별개의 의도이므로, UI 및 데이터 모델 모두 이를 분리하여 설계한다.

---

## 4. 적용 대상 기능 범위

| 기능 | 적용 원칙 |
|------|----------|
| **QR-code 제작** | 제품/채널 정보를 직접 선택 후 QR 생성. 결과 이미지만 저장 선택 가능 |
| **POP 제작** | 제조사 제품 자료·이미지를 직접 불러와 POP 레이아웃에 적용 |
| **디지털사이니지** | 공용 콘텐츠를 플레이리스트에 직접 편성. 플레이리스트 자체는 매장 자산으로 저장 |
| **블로그 글 제작** | 공용 자료·상품 정보를 에디터에 직접 삽입. 발행된 글이 저장 단위 |
| **타블렛/전자상거래용 상품 정보** | 공급자 상품 정보를 직접 참조하여 매장 진열 설정. 진열 설정이 저장 단위 |

---

## 5. 예외 — 내 매장 저장이 필요한 경우

다음 경우에는 내 매장 자산으로의 명시적 저장이 정당하다.

1. **반복 사용하는 결과물** — 동일한 POP·QR을 여러 채널에서 재사용할 때
2. **매장 전용 편집본** — 공용 자료를 이 매장 고유의 정보로 수정한 결과물
3. **플레이리스트·템플릿** — 편성 순서·레이아웃 구성 자체가 매장의 운영 자산일 때
4. **승인 워크플로가 필요한 경우** — 본사 검수 등 상태 추적이 필요한 자료

---

## 6. 후속 조사·리팩토링에서의 판단 기준

다음 질문을 기준으로 기존 구현의 흐름을 판단한다.

**Q1. 이 기능에서 "내 매장 가져오기" 단계가 없으면 작업이 불가한가?**
- 불가 → 직접 불러오기 흐름으로 리팩토링 필요
- 가능 → 현행 유지 또는 단계 제거 검토

**Q2. 내 매장에 저장되는 것이 "공용 자료의 복사본"인가 "작업 결과물"인가?**
- 복사본이 주 → 불필요한 중간 복사 제거 검토
- 결과물이 주 → 원칙 5(예외)에 해당, 저장 흐름 유지

**Q3. 공용 자료가 업데이트되었을 때 매장 화면이 자동 반영되어야 하는가?**
- 자동 반영 필요 → 복사 구조 금지, 직접 참조 구조 필수
- 매장 독립 관리 필요 → 복사본 저장 허용
