### 차단 사유 상세

**사유 1: 파일 형식**
- 파일 확장자: `.csv`
- 실제 형식: Excel Open XML (.xlsx) — PK 매직 바이트 + `[Content_Types].xml` 확인
- `csv-parse/sync` 파서로 파싱 불가

**사유 2: GTIN 부재**
```
전체 76개 컬럼 × 369행 스캔 결과:
GTIN-like 값 (8~14자리 숫자): 0건
→ 어떤 컬럼에도 산업 표준 바코드 없음
```

필수옵션바코드 컬럼의 실제 값: `a406`, `b656`, `a399a399`, `c039c136\r\nc039c137` — 모두 FirstMall 내부 상품코드

## 11. 권장 다음 단계

### 우선순위 1: 바코드 전략 결정

이것이 **모든 후속 작업의 전제 조건**이다.

| 선택지 | 설명 | 장점 | 단점 |
|--------|------|------|------|
| **A) GTIN 수동 부여** | 공급자가 실제 GTIN을 확보하여 CSV에 추가 | 산업 표준 준수 | 시간 소요, 소규모 공급자 부담 |
| **B) GTIN 정책 확장** | O4O Import에서 내부코드 허용 (GTIN 외 식별자) | 즉시 Import 가능 | 표준 바코드 체계 훼손 |
| **C) FirstMall 전용 변환기** | FirstMall 파서가 내부코드 → 가상 바코드 생성 | FirstMall 파서 안에서 해결 | 가상 바코드 관리 부담 |

### 우선순위 2: FirstMall → O4O 변환기

바코드 전략 결정 후, FirstMall Excel을 O4O CSV로 변환하는 전처리기 구현:

```
FirstMall Excel (.xlsx)
  ↓ 전처리기
  1. Excel → JSON 파싱
  2. 상품명 → marketing_name
  3. 추가정보 파싱 → manufacturer_name
  4. 매입가 or 정가 → supply_price
  5. 정가 → msrp
  6. 간략설명 → consumer_short_description
  7. 바코드 전략 적용 → barcode
  8. 이미지 URL 변환 → image_url (공개 URL 확보 후)
  ↓
O4O CSV 템플릿
  ↓ 기존 CSV Import
```

### 보류 가능

- 옵션 상품 Import 지원
- B2B 티어 가격 CSV 지원
- 상세 설명(HTML) Import
