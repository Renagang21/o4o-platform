# IR-O4O-SERVICE-CONFIG-O4OHELP-AUDIT-V1

## 목적

O4OHelpSection 텍스트의 service.config 적용 여부만 판단한다.

---

## 결론

```
O4OHelpSection → config 적용 대상 아님
service.config 확장하지 않는다
```

---

## 판단 근거

| 기준 | 결과 |
|------|------|
| 서비스별 텍스트 차이 반복 | ❌ 거의 없음 |
| UI 구조 동일 | ✔ |
| 도메인 로직 없음 | ✔ |

핵심 조건(서비스별 차이 반복)이 불충족 → 적용 불필요

---

## 이유

- "매장 등록 / 상품 등록 / 콘텐츠 활용" — KPA/Glyco/KCos 모두 동일 또는 거의 동일
- 공통 문구는 공통으로 두는 것이 맞음
- config로 빼면 의미 없는 추상화, 유지보수 오히려 어려움

---

## 예외 가능성

"매장 등록" vs "약국 등록" 차이가 있다면 고려 가능.
현재는 실제 서비스에서 동일하게 사용 가능하므로 적용 불필요.

---

## 현재 service.config 상태

```
✔ store 계열 반복 텍스트 → config 완료
✔ Home / Store / Hub → 적용 완료
✔ 불필요 영역 → 적용 안 함
```

---

## 원칙

> config는 많이 쓰는 게 아니라 "필요한 곳에만 쓰는 것"이 정답이다
