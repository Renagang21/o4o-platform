## 1. 작업 목적 (Why)

본 Work Order의 목적은 SiteGuide를 도입하려는 **외부 사업자**와의 협업을 전제로,

1. **어떤 사업자가**
2. **어떤 조건으로**
3. **어디까지의 기능과 책임 범위 안에서**
4. **SiteGuide를 사용할 수 있는지**

를 **기술 구현 이전에 명확히 고정**하는 것이다.

본 작업은 **영업 문서가 아니라, 개발 범위를 고정하기 위한 온보딩 설계 문서**이며,
이 문서를 기준으로 이후 모든 기술·UI·위젯 개발이 진행된다.

## 2. SiteGuide 사업자 온보딩의 기본 원칙

아래 원칙은 모든 사업자에게 **공통 적용**되며, 예외는 없다.

### 2.1 핵심 원칙 5가지

| # | 원칙 | 설명 |
|---|------|------|
| 1 | **외부 부가 서비스** | 사업자 홈페이지의 CMS·서버·DB를 변경하지 않는다 |
| 2 | **광고 플랫폼 아님** | 광고 기능은 선택 사항이며, 기본값은 OFF |
| 3 | **개인정보 비수집** | 개인정보 기반 타기팅을 하지 않는다 |
| 4 | **안내·보조 도구** | 해석·판단 도구가 아니라, 안내·보조 도구이다 |
| 5 | **공용 서비스** | 여러 사업자가 동시에 사용하며, 특정 사업자 전용 요구는 기본 구조에 포함되지 않는다 |

## 3. 온보딩 대상 사업자 유형 정의

사업자는 SiteGuide 온보딩 시 아래 유형 중 하나로 분류된다.

### 3.1 유형 A – 정보 중심 사업자 (Information-driven)

**예시**
- 언론·미디어
- 협회·학회
- 공공·비영리 기관
- 브랜드 소개 사이트

**특징**
| 항목 | 값 |
|------|-----|
| 광고 기능 | ❌ 사용 안 함 |
| 주요 목적 | 정보 보조·안내 |
| 중시 가치 | 신뢰도·중립성 |

**SiteGuide 제공 범위**
- 페이지 맥락 기반 AI 대화
- 정보성 Context Asset
- 최소 집계(호출 수, 노출 수)

---

### 3.2 유형 B – 상업 중심 사업자 (Commercial-driven)

**예시**
- 커머스
- 브랜드/제조사
- 서비스 판매 사이트

**특징**
| 항목 | 값 |
|------|-----|
| 광고 기능 | ⭕ 선택 가능 |
| 주요 목적 | 내부 상품·서비스 홍보 |
| 중시 가치 | 전환·노출 |

**SiteGuide 제공 범위**
- 페이지 맥락 기반 AI 대화
- 정보 + 상업 제안 Context Asset
- 노출/클릭 단위 집계

---

### 3.3 유형 C – 혼합형 사업자 (Hybrid)

**예시**
- 콘텐츠 + 상품 판매 병행
- 미디어 커머스
- 플랫폼형 서비스

**특징**
| 항목 | 값 |
|------|-----|
| 광고 기능 | ⭕ ON/OFF 선택 가능 |
| 제어 단위 | 페이지/섹션별 제어 가능 |
| 중시 가치 | 유연성 |

**SiteGuide 제공 범위**
- 유형 A + B의 조합
- 단, **광고 노출 규칙은 명시적으로 고정**

## 4. 제공 기능 범위 (기본 / 선택)

### 4.1 기본 제공 기능 (모든 사업자 공통)

| 기능 | 설명 |
|------|------|
| AI 질의 응답 | 페이지 맥락 기반 |
| 진입 방식 | 플로팅/오버레이 방식 |
| Context Asset | 정보성 노출 |
| 집계 데이터 | 페이지별 AI 호출 수, 제안 발생 여부, 민감 맥락 차단 횟수 |

> **이 범위는 필수이며, 제외 불가**

---

### 4.2 선택 제공 기능 (사업자 선택)

| 기능 | 설명 | 기본값 |
|------|------|--------|
| 상업적 제안 | Context Asset (conversion/branding) | OFF |
| 클릭/요청 집계 | 상세 이벤트 추적 | OFF |
| 광고 OFF + 집계 ON | 분리 모드 | OFF |

> 선택 기능은 **온보딩 시 명시적으로 선택**해야 하며,
> 운영 중 임의 변경 불가 (재합의 필요)

---

## 5. 광고·상업 제안 관련 온보딩 규칙

### 5.1 광고 기본 정책

| 규칙 | 설명 |
|------|------|
| 기본값 | **OFF** |
| 타기팅 방식 | 개인정보 기반 ❌, 페이지 맥락 기반만 ⭕ |
| 분리 원칙 | AI 응답과 광고/제안은 명확히 분리 |
| UX 원칙 | "광고처럼 보이면 실패" |

### 5.2 SiteGuide의 광고 책임 범위

| 항목 | SiteGuide 책임 |
|------|---------------|
| 광고 성과 해석 | ❌ 하지 않음 |
| 집계 제공 | ⭕ 단순 수치만 |
| 최적화 제안 | ❌ 하지 않음 |

## 6. 데이터 제공 및 보존 원칙

### 6.1 SiteGuide가 수집하는 데이터

| 데이터 | 목적 |
|--------|------|
| 페이지 메타데이터 | 카테고리, 태그 등 맥락 파악 |
| AI 호출 이벤트 | 사용량 집계 |
| 제안 노출/클릭 이벤트 | 선택 기능 집계 |
| 시스템 로그 | 안정성 목적 |

### 6.2 수집하지 않는 데이터

| 데이터 | 이유 |
|--------|------|
| 사용자 실명, 연락처 | 개인정보 비수집 원칙 |
| 결제 정보 | 범위 외 |
| 회원 DB 접근 | 범위 외 |
| 사업자 내부 데이터베이스 | 외부 부가 서비스 원칙 |

### 6.3 데이터 보존 및 활용

| 항목 | 방식 |
|------|------|
| 집계 데이터 | 요약 수치 중심 |
| 세밀한 분석 | AI 요약 기능으로 제공 |
| 원문 대화 데이터 | 기본 저장 ❌ |

## 7. 책임과 경계 (계약 기준 문장)

### 7.1 공식 책임 선언

아래 문장은 온보딩 시 **반드시 명시**된다:

> **SiteGuide 책임 경계 선언**
>
> 1. SiteGuide는 **안내 및 보조 도구**이며, 콘텐츠 및 상업적 제안의 최종 책임은 **사업자에게 있다.**
>
> 2. SiteGuide는 **편집권을 가지지 않으며**, 사업자의 의사결정을 대체하지 않는다.
>
> 3. SiteGuide 장애는 사업자 홈페이지의 **핵심 서비스 장애로 간주되지 않는다.**
>
> 4. AI 응답의 정확성은 **보장되지 않으며**, 참고 정보로만 제공된다.

### 7.2 책임 귀속 매트릭스

| 항목 | 책임 주체 |
|------|----------|
| AI 응답 내용 정확성 | 면책 (참고 정보) |
| 사이트 콘텐츠 적절성 | **사업자** |
| 상업 제안 내용 | **사업자** |
| 위젯 기술 안정성 | **SiteGuide** |
| 집계 데이터 정확성 | **SiteGuide** |

## 8. 온보딩 완료 기준 (Definition of Onboarding Ready)

아래 조건이 충족되면 해당 사업자는 **기술 연동 단계로 이동 가능**하다.

### 8.1 필수 체크리스트

- [ ] 사업자 유형(A/B/C) 확정
- [ ] 광고 ON/OFF 선택 확정
- [ ] 제공 기능 범위 합의
- [ ] 책임·데이터 경계 문장 합의

> **이 단계 이전에는 위젯/스크립트 개발 착수 불가**

### 8.2 온보딩 프로세스 흐름

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  사업자     │     │   유형      │     │   기능      │     │   책임      │
│  접촉       │ ──> │   분류      │ ──> │   범위      │ ──> │   합의      │
│             │     │  (A/B/C)    │     │   확정      │     │             │
└─────────────┘     └─────────────┘     └─────────────┘     └─────────────┘
                                                                   │
                                                                   ▼
                                                           ┌─────────────┐
                                                           │  기술 연동  │
                                                           │  단계 진입  │
                                                           └─────────────┘
```
