### 핵심 원칙

```
서비스 ─(진입 링크)→ Neture B2B 조달 ─(거래)→ 공급자

"서비스는 안내, Neture는 거래"
```

## 2. 구매자 자격 규칙

### 2.1 공통 요건

| 항목 | 필수 여부 | 설명 |
|------|----------|------|
| 사업자등록번호 | 필수 | 국세청 진위확인 |
| 사업자 유형 | 필수 | 일반/약국/의료기관 등 |
| 대표자명 | 필수 | 사업자등록증 기준 |
| 사업장 주소 | 필수 | 배송 및 세금계산서용 |
| 세금계산서 이메일 | 필수 | 전자세금계산서 수신 |

### 2.2 유형별 추가 요건

| 구매자 유형 | 추가 요건 | 검증 방법 | 접근 범위 |
|------------|----------|----------|----------|
| 일반 사업자 | 없음 | 사업자등록 확인 | 일반 상품 |
| 약국 | 약국 개설등록증 | 심평원 API / 문서 확인 | 일반 + 의약품 |
| 의료기관 | 의료기관 개설신고증 | 문서 확인 | 일반 + 의료용품 |

### 2.3 자격 상태

| 상태 | 설명 | 접근 |
|------|------|------|
| `pending` | 심사 대기 | 열람 불가 |
| `verified` | 인증 완료 | 자격 범위 내 접근 |
| `suspended` | 일시 정지 | 열람 불가 |
| `rejected` | 거부됨 | 재신청 필요 |

## 3. 상품 접근 규칙

### 3.1 상품 카테고리

| 대분류 | 중분류 | 접근 자격 |
|--------|--------|----------|
| 화장품 | 기초화장품 | 일반 사업자 |
| 화장품 | 기능성화장품 | 일반 사업자 |
| 약국제품 | 일반의약품 | 약국 전용 |
| 약국제품 | 의약외품 | 일반 사업자 |
| 약국제품 | 건강기능식품 | 일반 사업자 |
| 공통소모품 | 사무용품 | 일반 사업자 |
| 공통소모품 | 포장재 | 일반 사업자 |

### 3.2 상품 속성

| 속성 | 타입 | 설명 |
|------|------|------|
| `requiredBuyerType` | enum[] | 구매 가능 자격 유형 |
| `taxType` | enum | 과세/면세/영세 |
| `minOrderQty` | number | 최소 주문 수량 |
| `serviceExposure` | boolean | 서비스 노출 허용 여부 |
| `contentIds` | string[] | 연결된 콘텐츠 ID |

### 3.3 접근 제어 로직

```
상품 접근 가능 여부 =
  구매자.자격상태 === 'verified'
  AND 구매자.유형 IN 상품.requiredBuyerType
```

## 4. 서비스 연계 원칙

### 4.1 역할 분리

| 주체 | 역할 | 예시 |
|------|------|------|
| 서비스 | 진입점 제공 | "B2B 조달 바로가기" 링크 |
| 서비스 | 맥락 안내 | "약국 운영에 필요한 상품" |
| 서비스 | 추천 카테고리 | 서비스별 관련 카테고리 하이라이트 |
| Neture | 자격 검증 | 사업자/약국 확인 |
| Neture | 거래 처리 | 주문, 기록, 증빙 |
| Neture | 공급자 연결 | 상품/콘텐츠 제공 |

### 4.2 서비스별 진입 설정

| 서비스 | 추천 카테고리 | 진입 문구 |
|--------|--------------|----------|
| Yaksa | 약국제품, 화장품 | "약국 운영에 필요한 상품 조달" |
| Glycopharm | 건강기능식품, 의료용품 | "당뇨 관리 관련 상품 조달" |
| (공통) | 전체 | "B2B 상품 조달" |

### 4.3 금지 사항

| 금지 | 이유 |
|------|------|
| 서비스별 독자 거래 화면 | 중복/불일치 방지 |
| 서비스별 가격 차등 | 공정성 |
| 서비스별 별도 정산 | 단일 거래 원장 |

## 5. 거래 흐름

### 5.1 기본 흐름

```
[진입]
  │
  ├─ 서비스 → "B2B 조달" 링크 클릭
  │     또는
  └─ Neture 메인 → 로그인
         │
         ▼
[자격 검증]
  │
  ├─ 미인증 → 사업자 등록 유도
  │
  └─ 인증됨 → 자격 범위 확인
         │
         ▼
[상품 열람]
  │
  ├─ 카테고리 탐색
  │
  ├─ 상품 상세 확인
  │     └─ 공급자 콘텐츠 (설명/교육/마케팅)
  │
  └─ 접근 불가 상품 → 자격 안내 표시
         │
         ▼
[주문 요청]
  │
  ├─ 수량 입력
  │
  └─ 주문 확정
         │
         ▼
[증빙 생성]
  │
  ├─ 거래 기록 저장
  │
  └─ 세금계산서 기준 데이터 생성
```

### 5.2 상태 흐름

```
주문요청 → 접수완료 → 처리중 → 출고완료 → 거래완료
                         │
                         └─ 취소요청 → 취소완료
```
