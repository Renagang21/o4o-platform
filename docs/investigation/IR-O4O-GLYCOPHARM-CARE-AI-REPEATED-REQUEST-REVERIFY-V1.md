### 분석

- **서버 측**: 동일 토큰+메시지로 curl 호출 시 200 → **서버 API 자체는 정상**
- **프론트엔드 측**: 브라우저에서만 404 → **프론트엔드 요청 구성 문제 가능성**
- 추정 원인:
  1. 브라우저 세션 상태가 특정 횟수 이후 요청 URL 구성에 영향
  2. 스트림 연결 관리에서 이전 연결 정리 미완료 시 URL 오류
  3. Service Worker 또는 HTTP 캐시 레이어에서 잘못된 404 반환

### 수정 후 개선 확인

- 이전: 다양한 원인이 `AI chat processing failed`로 은폐됨
- 현재: 404 발생 시 "Request failed"로 표시 → **아직 generic**

### 스트리밍 상세

curl 직접 호출로 확인한 스트리밍 구조:
```
event: token
data: { "summary": "...", "details": [...] ... }  (청크 단위)

event: complete
data: { 전체 JSON }

event: done
data:
```

- 토큰 단위 스트리밍이 실제 동작 중 (sync fallback 아님)
- 응답 구조: summary + details[] + recommendations[] + relatedPatients[] + actions[]
