> 조사일: 2026-02-26 | 코드 수정: 없음
> 선행 문서: Phase1(라우트/메뉴/가드), Phase2(Care 구현 성숙도)

## 요약

요구사항 6대 메뉴와 현재 구현 상태를 비교한 결과:

| 구분 | 수량 |
|------|------|
| 재배치로 해결 가능 | 2건 |
| 프론트 연동만 필요 (백엔드 존재) | 5건 |
| 신규 개발 필요 | 8건 |
| 사업자 확인 필요 | 6건 |

**가장 큰 Gap:** 4대 축 중 3축(생활/복약/신체건강)이 백엔드·프론트 모두 전무하며,
혈당 축도 백엔드는 있으나 프론트 연동이 안 된 상태.

**심각도:** HIGH — 개발 범위/일정에 가장 큰 영향

**판단 필요:**

**심각도:** MEDIUM — CMS 활용 시 낮은 개발 비용, 독립 시 높은 비용

**심각도:** MEDIUM — 라우트 구조 정리 필요

### D6. CareDashboardPage의 "서비스 운영·관리" 확장 범위
