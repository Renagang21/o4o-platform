## 발견된 문제

### 1. 건강기능식품 카테고리 규제 필드 미표시 (WARN)

**현상**: 건강기능식품 카테고리 선택 시 `regulatoryName`(규제명)과 `mfdsPermitNumber`(허가번호) 입력 필드가 나타나지 않음. `regulatoryType` select만 1개 발견.

**비교**: 의약외품 카테고리에서는 `regulatoryType` select 2개 + `regulatoryName` + `mfdsPermitNumber` 필드 정상 표시.

**원인 추정**:
- `ProductCategory.isRegulated` 값이 건강기능식품에 `false`로 설정되어 있을 가능성
- 또는 카테고리 선택 후 UI 렌더링 타이밍 문제

**영향**: 건강기능식품 상품의 규제명/허가번호 미입력 상태로 등록됨. API 수준에서 `regulatoryType`은 설정되었으나 `regulatoryName`/`mfdsPermitNumber`는 누락.

**후속**: 건강기능식품 카테고리의 `isRegulated` 플래그 확인 필요.

### 2. 의료기기 카테고리 부재 (INFO)

**현상**: Seed 기준 "의료기기" 정확한 카테고리가 없음. "건강기기"(혈당/체온/혈압)와 "의료소모품"은 있지만 "의료기기"는 부재.

**처리**: 의료기기 상품을 일반상품에 매핑하여 등록. `regulatoryType=MEDICAL_DEVICE`는 설정.

**후속**: 의료기기 카테고리 추가 또는 건강기기/의료소모품으로 매핑 정책 결정 필요.

### 3. 이미지 등록 미실행 (INFO)

**현상**: 이번 스크립트에서는 상품 생성(3-step wizard)만 실행. 이미지 업로드/라이브러리 선택은 미실행.

**사유**: 이미지 업로드는 별도 API(`POST /products/:masterId/images`)를 통해 이루어지며, 이번 WO의 핵심 목표인 "상품 등록 흐름 검증"과는 분리. 이전 WO(IR-NETURE-IMAGE-LIBRARY-AND-EDITOR-FLOW-BROWSER-VERIFY-V1)에서 이미지 기능은 별도 검증 완료.

### 4. 1차 실행 중복 데이터 (INFO)

**현상**: 스크립트 디버깅 과정에서 1차 실행(6개)이 먼저 실행되어 총 26개의 테스트 상품이 존재할 수 있음.

**식별**: 중복 상품은 다른 `masterId`를 가지므로 식별 가능. 상품명으로 검색하면 동일 이름 상품이 2개씩 표시될 수 있음.
