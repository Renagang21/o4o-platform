# Legacy 대시보드 제거 계획서

*(Phase-out Plan - 통합 대시보드 v1 기준)*

**Status: Active**
**Created: 2026-01-10**
**Version: 1.0**

---

## 0. 문서 목적

본 계획서는 O4O Platform에 잔존하는 **Legacy 대시보드들을 체계적으로 종료(Phase-out)** 하기 위한 기준을 정의한다.

목표는 다음과 같다.

* 통합 대시보드 v1을 **유일한 기본 진입점**으로 고정
* 중복/혼란/운영 비용을 유발하는 화면 제거
* 사용자/운영자 혼란 없이 단계적 종료
* "혹시 필요할지 몰라 남겨둔 화면"을 명확히 정리

---

## 1. 기본 원칙 (고정)

1. **신규 기능은 Legacy에 절대 추가하지 않는다**
2. Legacy는 "참고용" 상태만 유지한다
3. 모든 대체 기능은 **통합 대시보드 카드로 이미 제공**된다
4. 제거는 **단계적으로**, 하지만 **되돌릴 수 없게** 진행한다

---

## 2. Legacy 대시보드 정의

### 2.1 Legacy 대시보드란?

다음 조건 중 하나라도 해당하면 Legacy로 간주한다.

* 통합 대시보드 v1 이전에 생성됨
* 역할(Role) 기준으로 분기된 대시보드
* 현재는 카드로 대체 가능한 정보만 제공
* `/dashboard/legacy` 이하로 이동된 화면

---

### 2.2 현재 Legacy 대상 목록 (확정)

| 경로                       | 기존 역할    | 대체 상태        |
| ------------------------ | -------- | ------------ |
| `/dashboard/legacy`      | user     | 통합 대시보드      |
| `/sellerops/dashboard`   | seller   | SellerCard   |
| `/supplierops/dashboard` | supplier | SupplierCard |
| `/partnerops/dashboard`  | partner  | PartnerCard  |
| `/membership/dashboard`  | operator | OperatorCard |

> `/admin/*`, `/yaksa-admin/*`, `/monitoring/*` 은 **Legacy 아님**
> (전문 운영 도구로 유지)

---

## 3. Phase-out 단계 정의

### Phase 0 - 현재 상태 (완료)

* 통합 대시보드 v1 정식 운영
* Legacy 화면 분리(`/dashboard/legacy`)
* Deprecated UI 표시 완료

---

### Phase 1 - 제한 유지 단계 (권장: 1~2개월)

**목적**

* 사용자 적응
* 내부 검증
* 긴급 롤백 가능성 확보

**정책**

* Legacy 접근 가능 O
* 신규 기능 추가 X
* 버그 수정 O (치명적 오류만)
* 통합 대시보드로 이동 CTA 항상 노출

---

### Phase 2 - 접근 제한 단계

**정책**

* Legacy 직접 진입 차단
* 기존 URL 접근 시:
  * 통합 대시보드로 리디렉션
  * "이 화면은 종료되었습니다" 안내 표시
* 운영자/관리자만 강제 접근 가능 (디버그 목적)

---

### Phase 3 - 제거 단계 (Final)

**정책**

* Legacy 라우트 제거
* 컴포넌트/페이지 코드 삭제
* 테스트 코드/문서 정리
* 빌드 산출물에서 완전 제거

> 이 단계 이후에는 **되돌릴 수 없음**

---

## 4. 사용자 커뮤니케이션 기준

### 4.1 사용자 메시지 원칙

* "기능이 사라졌다" X
* "더 나은 통합 화면으로 이동했다" O

### 4.2 권장 문구 예시

> 이 화면은 더 이상 사용되지 않습니다.
> 모든 기능은 새로운 통합 대시보드에서 제공됩니다.

---

## 5. 기술적 제거 기준 (중요)

### 5.1 제거 조건 체크리스트

다음이 모두 충족되어야 Phase 3로 이동 가능하다.

- [ ] Seller/Supplier/Partner/Operator 카드 정상 동작
- [ ] 알림/AI 요약 통합 완료
- [ ] 임원 컨텍스트 카드 정상 노출
- [ ] Legacy 관련 긴급 버그 0건

---

### 5.2 삭제 대상 범위

* 라우터 엔트리
* 페이지 컴포넌트
* 관련 상태 관리 코드
* Legacy 전용 스타일/유틸

---

## 6. 운영 리스크 관리

| 리스크       | 대응               |
| --------- | ---------------- |
| 특정 사용자 불만 | Legacy 접근 기간 제공  |
| 기능 누락 인식  | 카드 매핑표 제공        |
| 내부 혼란     | 이 문서를 단일 기준으로 사용 |

---

## 7. 책임 및 결정 구조

* Legacy 관련 신규 요구사항 -> **반려**
* "이 화면 다시 필요" 요청 -> **카드로 재검토**
* 예외 승인 권한 -> **플랫폼 총괄**

---

## 8. 공식 종료 선언

> **Legacy 대시보드는 더 이상 기능 확장의 대상이 아니며,
> 통합 대시보드 v1을 기준으로
> 단계적으로 종료된다.**

---

## 9. 이 문서의 의미 (중요)

이 문서가 승인되는 순간부터:

* 대시보드 구조 논쟁 종료
* "임시로 남겨둔 화면" 개념 종료
* 통합 대시보드가 **플랫폼의 유일한 기준점**이 됨
