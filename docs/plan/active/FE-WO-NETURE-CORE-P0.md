# 🎨 Frontend Work Order

## FE-WO-NETURE-CORE-P0

**대상 Work Order**: WO-NETURE-CORE-V1
**목표**: Neture P0 화면 구현 (표현·흐름 중심, 데이터 최소)

---

## 0. 공통 전제 (반드시 준수)

* Design Core **v1.0 필수 적용**
* 관리 UI처럼 보이는 컴포넌트 ❌
* 버튼은 **외부 이동 또는 탐색용만 허용**
* 쓰기(Create/Update/Delete) UI ❌
* 모든 데이터는 **읽기 전용(Read-only)**

---

## 1. 공통 레이아웃 규칙

### 1.1 네비게이션

상단 글로벌 메뉴:

```
Home | 공급자 | 제휴 요청 | 콘텐츠
```

* "대시보드" 메뉴 ❌
* 로그인 상태에 따라 메뉴 변화 없음 (P0)

---

### 1.2 공통 UI 톤

* 카드 기반 레이아웃
* 데이터 테이블 ❌
* "관리 / 설정 / 승인" 단어 사용 ❌
* 문구는 **설명형, 중립적**

---

## 2. Home 화면 구현 지시

### 파일 예시

```
/pages/HomePage.tsx
```

---

### 2.1 Hero 섹션

**구성**

* Headline
  * "공급자를 찾고, 제휴를 연결하는 유통 정보 플랫폼"
* Subtext
  * "주문·결제 없이 조건과 기회를 투명하게 확인하세요"
* CTA 버튼 (2개)
  * [공급자 보기] → `/suppliers`
  * [제휴 요청 보기] → `/partners/requests`

❌ 애니메이션 과다
❌ 기능 설명 나열

---

### 2.2 공급자 미리보기 섹션

**구성**

* 공급자 카드 4~6개
* 카드 요소:
  * 로고
  * 공급자명
  * 대표 카테고리
* CTA:
  * "공급자 전체 보기"

👉 카드 클릭 시 `/suppliers/{slug}`

---

### 2.3 제휴 요청 미리보기 섹션 (핵심)

**섹션 타이틀**

> 제휴 파트너를 찾는 판매자

**카드 요소**

* 판매자명
* 서비스 유형 (badge)
* 제휴 대상 제품 수
* 제휴 기간
* 제휴 기준 수익 구조 (텍스트)
* CTA: "제휴 조건 보기"

👉 `/partners/requests/{id}`

---

## 3. 공급자 페이지 구현 지시

### 파일 예시

```
/pages/suppliers/[supplierSlug].tsx
```

---

### 3.1 상단 헤더

* 공급자 로고
* 공급자명
* 한 줄 소개
* 외부 연락 아이콘 버튼
  * Email / Phone / Web / Kakao

👉 클릭 시 외부 이동만 수행

---

### 3.2 공급자 소개 섹션

* 텍스트 기반 설명
* 이미지 사용 가능 (선택)

---

### 3.3 제품 리스트 섹션

**구성**

* 제품 카드 리스트
* 카드 요소:
  * 제품명
  * 카테고리
  * 간단 설명

❌ 가격 숫자 노출
❌ 장바구니 / 버튼

---

### 3.4 유통 조건 섹션

* 가격 정책: 설명 텍스트
* MOQ
* 배송 정책
  * 일반 / 도서 / 산간

---

### 3.5 하단 안내 문구

> "거래를 원하시면
> 각 서비스의 판매자 대시보드에서 신청하세요"

❌ 신청 버튼 금지
