# O4O-3-ROLE-FLOW-BASELINE-V1

> 이 문서는 O4O 3자 Canonical Flow의 기준 문서이다.
> 세부 구현 지시는 개별 WO 및 하위 기준 문서를 따른다.

## 문서 목적

O4O Platform에서 공급자 → 운영자 → 매장 경영자로 이어지는 3자 흐름의 책임 경계와 데이터 흐름을 정의한다.

## §1. 3자 역할 요약

| 역할 | 식별자 | 핵심 책임 |
|------|--------|---------|
| **공급자** | supplier | 원천 자료(제품 정보, 마케팅 자료) 제공 |
| **운영사업자** | operator | 자료 수신·등록·구성·매장 자산 제작·큐레이션 |
| **매장 경영자** | store_owner | HUB 자료 활용·내 매장 실행 자산 제작·오프라인 운영 |

## §2. Canonical Flow

```
공급자
 └─ 원천 자료 전달 (오프라인/업로드/제품 DB)
      ↓
운영사업자 (Operator)
 ├─ 자료 수신·등록·구성
 ├─ AI 활용 (초안·품질 보조)
 ├─ HUB 콘텐츠 게시 (상품 정보, POP, QR, 블로그, 사이니지 등)
 ├─ 매장 경영자 지원
 └─ 운영 수익 모델 구축
      ↓
매장 경영자 (Store Owner)
 ├─ HUB 자료에서 내 매장 실행 자산 복사/활용
 ├─ 내 매장 운영 (콘텐츠·사이니지·POP·QR)
 └─ 오프라인 고객 접점
      ↓
오프라인 고객
```

## §3. 책임 매트릭스

| 행위 | 공급자 | 운영자 | 매장 경영자 |
|------|:------:|:------:|:-----------:|
| O4O 내부 콘텐츠 직접 제작 | ❌ | ✅ | ✅ (내 매장 한정) |
| 공급자 원천 자료 제공 | ✅ | ❌ | ❌ |
| HUB 자료 게시·큐레이션 | ❌ | ✅ | ❌ |
| 매장 실행 자산 제작 | ❌ | 지원 | ✅ |
| 오프라인 고객 응대 | ❌ | ❌ | ✅ |
| 운영 정책·거버넌스 | ❌ | ✅ | ❌ |
| 재무·정산 책임 | 일부 | ✅ | ❌ |

## §4. 원천 자료 vs 실행 자산

| 구분 | 정의 | 예시 |
|------|------|------|
| **원천 자료** | 공급자가 전달하는 비가공 정보 | 제품 스펙, 성분표, 원본 이미지 |
| **실행 자산** | 운영자·매장 경영자가 만든 활용 가능한 콘텐츠 | POP, QR, 블로그 포스트, 사이니지 |

## §5. AI 개입 기준

AI는 운영자의 자료 처리 및 콘텐츠 초안 생성을 보조한다.

- **허용**: 운영자 Workspace B에서 초안 생성, 품질 향상
- **금지**: AI가 사람의 검수 없이 HUB에 직접 게시
- **금지**: AI가 매장 경영자 대신 내 매장 자산을 생성

## §6. Drift 금지 흐름

다음 흐름은 3자 역할 경계를 위반하며 금지된다.

- 공급자가 O4O 시스템에서 직접 HUB 콘텐츠를 제작·게시
- 운영자가 매장 경영자 대신 내 매장을 직접 운영 (예외는 명시적 WO 필요)
- 매장 경영자가 공급자에게 직접 요청 (운영자를 통해 처리)
- AI가 운영자 검수 없이 HUB에 게시

## §7. 권장 흐름 예시 (AI 개입 포함)

```
공급자 원천 자료
 → 운영자 등록 (Workspace A)
 → AI 초안 생성 (선택, Workspace B)
 → 운영자 검수
 → HUB 게시 (Workspace C)
 → 매장 경영자 복사
 → 내 매장 활용
```

AI 개입은 운영자 검수를 대체하지 않는다.
