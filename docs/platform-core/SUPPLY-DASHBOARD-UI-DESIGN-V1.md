# WO-O4O-SERVICE-OPERATOR-SUPPLY-DASHBOARD-UI-V1

## 서비스 운영자 대시보드 – 공급요청 UI 설계

> **참조 기준선**: `SUPPLY-REQUEST-BASELINE-V1.md`
> **성격**: UI/UX 설계 문서 (코드 구현 Out of Scope)

---

## 1. 목적

각 **서비스 운영자**가 자신의 **서비스 대시보드**에서
공급 가능 제품을 확인하고, 제품 단위로 **공급요청을 실행**할 수 있는
**화면 기준선(UI 계약)**을 정의한다.

---

## 2. 대상 사용자

- Service Operator (서비스 운영자)

비대상:
- Neture 관리자, 공급자, 판매자

---

## 3. 화면 위치

- 서비스 운영자 대시보드 내 메뉴
  - 예: **공급 관리 → 공급 가능 제품**
- Neture 접근 없이 **서비스 내부에서만** 동작

---

## 4. 데이터 범위

- 대상: 해당 서비스가 '공급 가능'으로 지정된 제품
- 제외: 다른 서비스 전용 제품, 서비스 미선택 제품

---

## 5. 상태 매핑 (기준선 준수)

| 상태 | UI 표시 | 버튼 |
|------|---------|------|
| 공급 가능 | 회색 배지 `공급 가능` | **[공급요청] 활성** |
| 요청됨 | 주황 배지 `요청됨` | 비활성 (중복 요청 불가) |
| 공급중 | 초록 배지 `공급중` | 비활성 |
| 거절됨 | 빨강 배지 `거절됨` | 비활성 + 사유(옵션) |

---

## 6. 리스트 UI 구성

카드 또는 테이블(경량):
- 제품명
- 공급자명
- 현재 상태 배지
- 주요 조건 (가격/카테고리 요약)
- 액션 영역

---

## 7. 액션 정의

### [공급요청] 버튼

- 노출 조건: 상태 = `공급 가능`
- 동작:
  - 즉시 요청 (확인 모달 1회)
  - 요청 성공 시 상태 → `요청됨`
- 추가 입력 불필요 (조건은 사전 합의 전제)

---

## 8. 알림/피드백

- 요청 성공 토스트
- 상태 변경 즉시 반영
- 실패 시 표준 에러 메시지

---

## 9. 비범위 (Out of Scope)

- 승인/거절 UI (공급자 영역)
- 가격 협상
- 계약 관리
- 알림 시스템 구현

---

## 10. 완료 기준

- [ ] 운영자 화면에서 "무엇을 보고 무엇을 누르는지"가 명확
- [ ] 기준선 상태와 1:1 매핑
- [ ] 코드/모델 변경 없이도 해석 가능
- [ ] 후속 구현 WO가 바로 작성 가능
